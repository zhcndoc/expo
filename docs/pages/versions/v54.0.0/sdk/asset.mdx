`expo-asset` 提供了一个访问 Expo 资源系统的接口。资源是指与你的应用源码放在一起、且应用在运行时需要的任何文件。例如图片、字体和声音文件。Expo 的资源系统与 React Native 的资源系统集成，因此你可以使用 `require('path/to/file')` 来引用文件。例如，在 React Native 中，你可以通过这种方式引用静态图片文件，并将其用于 `Image` 组件。有关更多信息，请查看 React Native 关于[静态图片资源的文档](https://reactnative.dev/docs/images#static-image-resources)。这种引用静态图片资源的方法在 Expo 中开箱即用。

## 在 app 配置中的配置

如果你在项目中使用配置插件（[连续原生生成（CNG）](/workflow/continuous-native-generation/)），你可以使用其内置的[配置插件](/config-plugins/introduction/)来配置 `expo-asset`。该插件允许你配置一些无法在运行时设置、且需要构建新的应用二进制文件后才会生效的属性。如果你的应用**没有**使用 CNG，那么你需要手动配置该库。

<ConfigPluginExample>

```json app.json
{
  "expo": {
    "plugins": [
      [
        "expo-asset",
        {
          "assets": ["path/to/file.png", "path/to/directory"]
        }
      ]
    ]
  }
}
```

</ConfigPluginExample>
