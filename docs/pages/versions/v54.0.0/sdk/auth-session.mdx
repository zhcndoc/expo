`AuthSession` 通过利用 [WebBrowser](webbrowser.mdx) 和 [Crypto](crypto.mdx)，使你的应用能够基于网页浏览器进行身份验证（例如，基于浏览器的 OAuth 流程）。有关实现细节，请参考此文档；有关用法，请参阅 [Authentication](/guides/authentication/) 指南。

> **注意：** `AuthSession` 支持通用的 OAuth 和 OpenID Connect 基于浏览器的认证流程。在可用的情况下，我们建议使用身份提供商提供的库，因为它会处理该提供商特有的实现细节。例如，Google 认证请使用 [`@react-native-google-signin/google-signin`](/guides/google-authentication/)，Facebook 认证请使用 [`react-native-fbsdk-next`](/guides/facebook-authentication/)。更多信息请参阅 [Authentication](/develop/authentication/) 概览。

### 在独立应用中的用法

```json app.json
{
  "expo": {
    "scheme": "mycoolredirect"
  }
}
```

为了能够通过深度链接返回到你的应用，你需要在项目的 app 配置中设置一个 `scheme`，然后构建你的独立应用（它不能通过更新来修改）。如果你不包含 scheme，认证流程会完成，但它将无法把信息传回你的应用，用户将不得不手动退出认证弹窗（从而导致一个 canceled 事件）。

## 基于网页浏览器的认证流程如何工作

移动应用中基于浏览器的认证流程通常如下：

- **发起**：用户点击登录按钮
- **打开网页浏览器**：应用会打开一个网页浏览器，跳转到认证提供商的登录页面。打开的登录页面 URL 通常包含用于标识应用的信息，以及成功后重定向的 URL。_注意：网页浏览器应与系统浏览器共享 cookies，这样如果用户已经在系统浏览器中完成认证，就不需要再次登录——Expo 的 [WebBrowser](webbrowser.mdx) API 会负责这一点。_
- **认证提供商重定向**：认证成功后，认证提供商应通过重定向到登录页面查询参数中由应用提供的 URL，将用户重定向回应用（[了解更多关于移动应用中链接的工作方式](/linking/overview/)），_前提是该 URL 位于允许的重定向 URL 白名单中_。对重定向 URL 进行白名单管理很重要，可以防止恶意行为者冒充你的应用。重定向会在 URL 中包含数据（例如用户 id 和 token），可以位于 location hash、查询参数中，或两者都有。
- **应用处理重定向**：由应用处理该重定向，并从重定向 URL 中解析数据。

## 安全注意事项

- **切勿将任何密钥放入你的应用代码中，这没有任何安全方式！** 相反，你应该把密钥存储在服务器上，并提供一个端点，由它为你的客户端发起 API 调用并将数据返回。

## 高级用法

### 在 Linking 处理器中过滤掉 AuthSession 事件

你可能需要处理进入应用的链接，原因有很多，例如推送通知或常规深度链接（你可以在 [Linking](/linking/overview/) 中阅读更多相关内容）；认证重定向只是深度链接的一种，`AuthSession` 会为你处理这些特定链接。在你自己的 `Linking.addEventListener` 处理器中，可以通过检查 URL 是否包含 `+expo-auth-session` 字符串来过滤掉由 `AuthSession` 处理的深度链接——如果包含，就可以忽略它。这之所以有效，是因为 `AuthSession` 会在默认的 `returnUrl` 中添加 `+expo-auth-session`；不过，如果你提供了自己的 `returnUrl`，你可能也想考虑添加类似的标识符，以便从其他处理器中过滤掉 `AuthSession` 事件。

### 与 React Navigation 配合使用

如果你在 React Navigation 中使用深度链接，仅通过 `Linking.addEventListener` 进行过滤是不够的，因为深度链接的处理方式[不同](https://reactnavigation.org/docs/configuring-links/#advanced-cases)。相反，要过滤这些事件，请在你的 linking 配置中添加自定义的 `getStateFromPath` 函数，然后像上面所述那样按 URL 进行过滤。
