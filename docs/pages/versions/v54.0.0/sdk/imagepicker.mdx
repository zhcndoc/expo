`expo-image-picker` 提供了访问系统 UI 的能力，可用于从手机图库中选择图像和视频，或使用相机拍照。

#### 已知问题&ensp;<PlatformTags platforms={['ios']} />

在 iOS 上，当从相机胶卷中选择图像（通常是 [更高分辨率](http://www.openradar.me/49866214) 的图像）时，在某些情况下，裁剪后的图像结果会给出错误的裁剪矩形值。不幸的是，这个问题出在底层的 `UIImagePickerController`，原因是 iOS 内置闭源工具中的一个 bug。

## 在 app 配置中进行配置

如果你在项目中使用配置插件（[Continuous Native Generation (CNG)](/workflow/continuous-native-generation/)），你可以使用 `expo-image-picker` 内置的 [config plugin](/config-plugins/introduction/) 进行配置。该插件允许你配置各种无法在运行时设置的属性，这些属性需要重新构建新的应用二进制文件后才会生效。如果你的应用**没有**使用 CNG，那么你需要手动配置该库。

默认情况下，`expo-image-picker` 会在 Android 上添加 `RECORD_AUDIO` 权限。你可以通过将下面配置中的 `microphonePermission` 设为 false 来移除它。

## 用法

<SnackInline label='Image Picker' dependencies={['expo-image-picker']}>

```tsx
import { useState } from 'react';
import { Alert, Button, Image, View, StyleSheet } from 'react-native';
import * as ImagePicker from 'expo-image-picker';

export default function ImagePickerExample() {
  const [image, setImage] = useState<string | null>(null);

  const pickImage = async () => {
    // 启动图像库不需要请求权限。
    // 当 `allowsEditing` 设为 `false`
    // 且 `videoExportPreset` 为 `'Passthrough'`（默认值）时，在 iOS 上需要手动请求视频权限，最好在打开选择器之前请求，
    // 这样应用用户在选择视频后就不会被系统弹窗打扰。
    // 有关更多详细信息，请参阅“调用视频权限”子章节。
    const permissionResult = await ImagePicker.requestMediaLibraryPermissionsAsync();

    if (!permissionResult.granted) {
      Alert.alert('需要权限', '访问媒体库的权限是必需的。');
      return;
    }

    let result = await ImagePicker.launchImageLibraryAsync({
      mediaTypes: ['images', 'videos'],
      allowsEditing: true,
      aspect: [4, 3],
      quality: 1,
    });

    console.log(result);

    if (!result.canceled) {
      setImage(result.assets[0].uri);
    }
  };

  return (
    <View style={styles.container}>
      <Button title="从相机胶卷中选择图像" onPress={pickImage} />
      {image && <Image source={{ uri: image }} style={styles.image} />}
    </View>
  );
}

const styles = StyleSheet.create({
  container: {
    flex: 1,
    alignItems: 'center',
    justifyContent: 'center',
  },
  image: {
    width: 200,
    height: 200,
  },
});
```

</SnackInline>

### 调用视频权限 <PlatformTags platforms={['ios']} />

在 SDK 54 及更高版本中，默认配置会将 `allowsEditing` 保持为 `false`，并将 [`videoExportPreset`](#videoexportpreset) 设为 `'Passthrough'`。这些设置会因为选择器跳过压缩而立即返回原始资源（包括 HEIC 和 AVIF 文件），但 iOS 需要媒体库权限才能访问原始文件，并会在用户选择视频后立即显示权限对话框。

为了避免在选择后显示权限对话框，请在打开选择器之前通过 [`requestMediaLibraryPermissionsAsync`](#imagepickerrequestmedialibrarypermissionsasyncwriteonly) 或 [`useMediaLibraryPermissions`](#usemedialibrarypermissionsoptions) **手动请求媒体库权限**。
