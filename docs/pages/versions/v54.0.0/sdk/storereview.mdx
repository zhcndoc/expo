`expo-store-review` 是一个库，它提供了对 Android 5+ 上的 `ReviewManager` API 以及 iOS 上的 `SKStoreReviewController` API 的访问。它允许你在不离开应用本身的情况下，请求用户为你的应用评分。

## 用法

在使用此 API 时，务必遵循 iOS 的 [Human Interface Guidelines](https://developer.apple.com/design/human-interface-guidelines/ratings-and-reviews) 和 Android 的 [Guidelines](https://developer.android.com/guide/playcore/in-app-review#when-to-request)。

**具体来说：**

- 不要从按钮中调用 `StoreReview.requestReview()` - 而应尝试在用户完成应用中的某个标志性交互之后再调用。
- 不要频繁打扰用户。
- 当用户正在进行诸如导航之类的时间敏感操作时，不要请求评价。
- 在展示评分按钮或卡片之前或期间，不要询问用户任何问题。

### 撰写评论

#### Android

Android 上没有等效的重定向，不过你仍然可以使用查询参数 `showAllReviews=true` 打开 Play Store 的评论页面，如下所示：

```ts Android Play Store example
const androidPackageName = 'host.exp.exponent';
// 在浏览器中打开 Android Play Store -> 在 Android 上重定向到 Play Store
Linking.openURL(
  `https://play.google.com/store/apps/details?id=${androidPackageName}&showAllReviews=true`
);
// 直接打开 Android Play Store
Linking.openURL(`market://details?id=${androidPackageName}&showAllReviews=true`);
```

#### iOS

你可以通过使用查询参数 `action=write-review`，将应用用户重定向到 iOS App Store 中某个应用的 **“撰写评论”** 页面。例如：

```ts iOS App Store example
const itunesItemId = 982107779;
// 在浏览器中打开 iOS App Store -> 在 iOS 上重定向到 App Store
Linking.openURL(`https://apps.apple.com/app/apple-store/id${itunesItemId}?action=write-review`);
// 直接打开 iOS App Store
Linking.openURL(
  `itms-apps://itunes.apple.com/app/viewContentsUserReviews/id${itunesItemId}?action=write-review`
);
```

## 错误代码

### `ERR_STORE_REVIEW_FAILED`

当商店评论请求未成功时，会发生此错误。
