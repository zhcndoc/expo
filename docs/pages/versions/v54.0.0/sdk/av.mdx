> **warning** **已弃用：** 来自 `expo-av` 的 `Video` 和 `Audio` API 现已弃用，并已被改进版本替代，分别位于 [`expo-video`](video.mdx) 和 [`expo-audio`](audio.mdx)。我们建议改用这些库。`expo-av` 不再接收补丁，并将在 SDK 55 中移除。

[`Audio.Sound`](audio.mdx) 对象和 [`Video`](video-av.mdx) 组件在媒体播放方面共享一套统一的命令式 API。

请注意，对于 `Video`，所有操作也都可以通过组件上的 props 来使用。不过，对于大多数需要更细粒度控制视频播放状态的应用，我们建议使用这种命令式播放 API。

有关 `Audio.Sound` 和 `Video` 的播放 API 示例，请参阅 [播放列表示例应用](https://github.com/expo/playlist-example)。

> **Info** 音频录制 API 在 tvOS（Apple TV）上不可用。

## 在 app 配置中进行配置

如果你在项目中使用配置插件（[Continuous Native Generation (CNG)](/workflow/continuous-native-generation/)），可以使用 `expo-av` 内置的 [config plugin](/config-plugins/introduction/) 来进行配置。该插件允许你配置各种无法在运行时设置、且需要重新构建新的应用二进制文件才会生效的属性。如果你的应用**不**使用 CNG，那么你需要手动配置该库。

<ConfigPluginExample>

```json app.json
{
  "expo": {
    "plugins": [
      [
        "expo-av",
        {
          "microphonePermission": "Allow $(PRODUCT_NAME) to access your microphone."
        }
      ]
    ]
  }
}
```

如果你没有使用 Continuous Native Generation（[CNG](/workflow/continuous-native-generation/)）（也就是你在手动使用原生 **android** 和 **ios** 项目），那么你需要在原生项目中配置以下权限：

- 对于 Android，请将 `android.permission.RECORD_AUDIO` 权限添加到项目的 **android/app/src/main/AndroidManifest.xml** 中：

  ```xml
  <uses-permission android:name="android.permission.RECORD_AUDIO" />
  ```

- 对于 iOS，请将 `NSMicrophoneUsageDescription` 添加到项目的 **ios/[app]/Info.plist** 中：

  ```xml
  <key>NSMicrophoneUsageDescription</key>
  <string>Allow $(PRODUCT_NAME) to access your microphone</string>
  ```

</ConfigReactNative>

## 用法

在本页中，我们引用的是 `playbackObject` 上的操作。下面是一个获取音频和视频引用的示例：

### 示例：`Audio.Sound`

```js
await Audio.setAudioModeAsync({ playsInSilentModeIOS: true });

const playbackObject = new Audio.Sound();
// 或者
const { sound: playbackObject } = await Audio.Sound.createAsync(
  { uri: 'http://foo/bar.mp3' },
  { shouldPlay: true }
);
```

有关 `Audio.Sound.createAsync()` 的更多信息，请参阅 [音频文档](audio-av.mdx)。

### 示例：`Video`

```js
/* @hide ... */ /* @end */
_handleVideoRef = component => {
  const playbackObject = component;
  ...
}
/* @hide ... */ /* @end */

render() {
  return (
      <Video
        ref={this._handleVideoRef}
      />
      /* @hide ... */ /* @end */
  )

```

有关更多信息，请参阅 [视频文档](video-av.mdx)。

### 示例：`setOnPlaybackStatusUpdate()`

```js
_onPlaybackStatusUpdate = playbackStatus => {
  if (!playbackStatus.isLoaded) {
    // 为未加载状态更新你的 UI
    if (playbackStatus.error) {
      console.log(`播放过程中遇到致命错误：${playbackStatus.error}`);
      // 在 Slack 或论坛上把错误发送给 Expo 团队，这样我们就能帮你调试！
    }
  } else {
    // 为已加载状态更新你的 UI

    if (playbackStatus.isPlaying) {
      // 为播放状态更新你的 UI
    } else {
      // 为暂停状态更新你的 UI
    }

    if (playbackStatus.isBuffering) {
      // 为缓冲状态更新你的 UI
    }

    if (playbackStatus.didJustFinish && !playbackStatus.isLooping) {
      // 播放器刚刚完成播放并将停止。也许你想播放其他内容？
    }

    /* @hide ... */ /* @end */
  }
};

// 加载 playbackObject 并获取引用。
playbackObject.setOnPlaybackStatusUpdate(this._onPlaybackStatusUpdate);
```

### 示例：精确循环媒体 20 次

```js
const N = 20;
/* @hide ... */ /* @end */

_onPlaybackStatusUpdate = playbackStatus => {
  if (playbackStatus.didJustFinish) {
    if (this.state.numberOfLoops == N - 1) {
      playbackObject.setIsLooping(false);
    }
    this.setState({ numberOfLoops: this.state.numberOfLoops + 1 });
  }
};

/* @hide ... */ /* @end */
this.setState({ numberOfLoops: 0 });
// 加载 playbackObject 并获取引用。
playbackObject.setOnPlaybackStatusUpdate(this._onPlaybackStatusUpdate);
playbackObject.setIsLooping(true);
```

## 什么是 seek 容差，以及为什么我要使用它&ensp;<PlatformTags platforms={['ios']} />

当要求对一个 A/V 项目进行 seek 时，iOS 中的原生播放器有时可能会 seek 到一个略有不同的时间点。苹果文档中提到的这种技术（见 [Apple 文档](https://developer.apple.com/documentation/avfoundation/avplayer/1387741-seek#discussion)）用于缩短 `seekTo` 调用所需的时间（播放器可能会决定直接从与请求不同的时间点立即播放，而不是解码请求的精确部分并在解码延迟后播放）。

如果精确性很重要，你可以指定播放器 seek 时允许的容差。不过，这会导致延迟增加。
