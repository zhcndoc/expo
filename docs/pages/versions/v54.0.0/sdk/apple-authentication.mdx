`expo-apple-authentication` 为 iOS 提供 Apple 身份验证。它目前尚不支持 Android 或 web。

任何包含第三方身份验证选项的应用**必须**提供 Apple 身份验证作为一个选项，以符合 App Store 审核指南。更多信息请参阅 [使用 Apple 登录](https://developer.apple.com/sign-in-with-apple/) 网站上的 Apple 身份验证。

## 在 app 配置中进行配置

如果你在项目中使用 config plugins（[Continuous Native Generation (CNG)](/workflow/continuous-native-generation/)），你可以使用 `expo-apple-authentication` 内置的 [config plugin](/config-plugins/introduction/) 进行配置。该插件允许你配置各种无法在运行时设置、且需要构建新的应用二进制文件后才会生效的属性。如果你的应用**不**使用 CNG，那么你需要手动配置该库。

### 设置 iOS 项目

要在应用中启用 **使用 Apple 登录** 功能，请在项目的 app 配置中将 [`ios.usesAppleSignIn`](../config/app/#usesapplesignin) 属性设置为 `true`：

```json app.json
{
  "expo": {
    "ios": {
      "usesAppleSignIn": true
    }
  }
}
```

<ConfigPluginExample>

在本地运行 [EAS Build](/build/introduction) 时，会使用 [iOS capabilities signing](/build-reference/ios-capabilities) 在构建前启用所需的能力。

```json app.json
{
  "expo": {
    "plugins": ["expo-apple-authentication"]
  }
}
```

</ConfigPluginExample>

<ConfigReactNative>

不使用 [EAS Build](/build/introduction) 的应用必须为其 bundle identifier [手动配置](/build-reference/ios-capabilities#manual-setup) **Apple 登录** 能力。

如果你通过 [Apple Developer Console](/build-reference/ios-capabilities#apple-developer-console) 启用了 **Apple 登录** 能力，那么请务必在 **ios/[app]/[app].entitlements** 文件中添加以下 entitlements：

```xml
<key>com.apple.developer.applesignin</key>
<array>
  <string>Default</string>
</array>
```

另外，请在 **ios/[app]/Info.plist** 中将 `CFBundleAllowMixedLocalizations` 设置为 `true`，以确保登录按钮使用设备语言区域。

</ConfigReactNative>

## 使用

<SnackInline label="Apple Authentication Usage" dependencies={['expo-apple-authentication']} platforms={['ios']}>

```jsx
import * as AppleAuthentication from 'expo-apple-authentication';
import { View, StyleSheet } from 'react-native';

export default function App() {
  return (
    <View style={styles.container}>
      <AppleAuthentication.AppleAuthenticationButton
        buttonType={AppleAuthentication.AppleAuthenticationButtonType.SIGN_IN}
        buttonStyle={AppleAuthentication.AppleAuthenticationButtonStyle.BLACK}
        cornerRadius={5}
        style={styles.button}
        onPress={async () => {
          try {
            const credential = await AppleAuthentication.signInAsync({
              requestedScopes: [
                AppleAuthentication.AppleAuthenticationScope.FULL_NAME,
                AppleAuthentication.AppleAuthenticationScope.EMAIL,
              ],
            });
            // 已登录
          } catch (e) {
            if (e.code === 'ERR_REQUEST_CANCELED') {
              // 处理用户取消登录流程
            } else {
              // 处理其他错误
            }
          }
        }}
      />
    </View>
  );
}

const styles = StyleSheet.create({
  container: {
    flex: 1,
    alignItems: 'center',
    justifyContent: 'center',
  },
  button: {
    width: 200,
    height: 44,
  },
});
```

</SnackInline>

## 开发与测试

你可以在 iOS 上的 Expo Go 中测试这个库，而无需遵循以上任何说明。
不过，如果你正在使用 EAS Build，则需要添加 config plugin 才能使用这个库。
当你登录 Expo Go 时，你收到的标识符和值很可能会与在独立应用中收到的不同。

你可以在 iOS 模拟器上对这个库进行有限的测试。不过，并非所有方法的表现都与真机相同，
因此我们强烈建议在开发过程中尽可能使用真机进行测试。

## 验证来自 Apple 的响应

Apple 的响应包含一个带有用户信息的签名 JWT。为确保该响应来自 Apple，
你可以使用 Apple 的公钥对签名进行加密验证，该公钥发布于 https://appleid.apple.com/auth/keys。
此过程并非 Expo 特有。

## 错误代码

大多数错误代码与官方的 [Apple Authorization errors](https://developer.apple.com/documentation/authenticationservices/asauthorizationerror/code) 相匹配。

| 代码                        | 描述                                                                             |
| --------------------------- | -------------------------------------------------------------------------------- |
| ERR_INVALID_OPERATION       | 执行了无效的授权操作。                                                            |
| ERR_INVALID_RESPONSE        | 授权请求收到了无效响应。                                                          |
| ERR_INVALID_SCOPE           | 传入了无效的 [`AppleAuthenticationScope`](#appleauthenticationscope)。           |
| ERR_REQUEST_CANCELED        | 用户取消了授权尝试。                                                              |
| ERR_REQUEST_FAILED          | 授权尝试失败。有关更多信息，请参阅错误消息。                                      |
| ERR_REQUEST_NOT_HANDLED     | 授权请求未被正确处理。                                                            |
| ERR_REQUEST_NOT_INTERACTIVE | 授权请求不是交互式的。                                                            |
| ERR_REQUEST_UNKNOWN         | 授权尝试因未知原因失败。                                                          |
