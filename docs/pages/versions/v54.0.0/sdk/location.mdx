## 在 app 配置中进行配置

如果你在项目中使用 config plugins（[Continuous Native Generation (CNG)](/workflow/continuous-native-generation/)），可以使用 `expo-location` 内置的 [config plugin](/config-plugins/introduction/) 来进行配置。该插件允许你配置一些无法在运行时设置、且需要构建新的应用二进制文件后才会生效的属性。如果你的应用**不**使用 CNG，那么你需要手动配置该库。

### 后台位置

后台位置允许你的应用在后台运行时接收位置更新，包括通过地理围栏进行的位置更新和区域监测。该功能受平台 API 限制和系统约束的影响：

- 如果用户终止应用，后台位置将停止。
- 如果用户重新启动应用，后台位置会恢复。
- <PlatformTag platform="android" /> 由于平台限制，已终止的应用在发生位置
  或地理围栏事件时不会自动重启。
- <PlatformTag platform="ios" /> 当发生新的地理围栏
  事件时，系统会重新启动已终止的应用。

> **信息** 在 Android 上，从最近应用列表中移除一个应用的结果因设备厂商而异。例如，某些实现会将从最近应用列表中移除应用视为杀死该应用。有关这些差异的更多信息，请参见： [https://dontkillmyapp.com](https://dontkillmyapp.com)。

### 后台位置配置&ensp;<PlatformTag platform="ios" />

要在 iOS 上运行后台位置，你需要在应用的 **Info.plist** 文件中的 `UIBackgroundModes` 数组里添加 `location` 值。

**如果你正在使用 [CNG](/workflow/continuous-native-generation/)**，所需的 `UIBackgroundModes` 配置将通过 prebuild 自动应用。

<Collapsible summary="在 iOS 上手动配置 UIBackgroundModes">

如果你没有使用 Continuous Native Generation ([CNG](/workflow/continuous-native-generation/))，或者你正在使用原生 iOS 项目，那么你需要将以下内容添加到你的 **Expo.plist** 文件中：

```xml ios/project-name/Supporting/Expo.plist
<key>UIBackgroundModes</key>
  <array>
    <string>location</string>
  </array>
```

</Collapsible>

### 后台位置方法

要使用后台位置方法，需要满足以下要求：

- 必须授予位置权限。
- 必须在顶层作用域中使用 [`TaskManager.defineTask`](task-manager.mdx#taskmanagerdefinetasktaskname-taskexecutor) 定义后台位置任务。
- <PlatformTag platform="ios" className="float-left" /> **Info.plist** 文件中必须指定 `"location"` 后台模式。请参见[后台位置
  配置](#background-location-configuration)。
- <PlatformTag platform="ios" className="float-left" /> 你必须使用 [development
  build](/develop/development-builds/introduction/) 才能使用后台位置，因为 Expo Go 应用不支持它。

### 地理围栏方法

要使用地理围栏方法，需要满足以下要求：

- 必须授予位置权限。
- 必须在顶层作用域中使用 [`TaskManager.defineTask`](task-manager.mdx#taskmanagerdefinetasktaskname-taskexecutor) 定义地理围栏任务。

使用地理围栏时，适用以下平台差异：

- <PlatformTag platform="android" /> 每个应用最多允许 [100 个](https://developer.android.com/develop/sensors-and-location/location/geofencing) 活跃
  的地理围栏。
- <PlatformTag platform="ios" /> Expo Location 会在应用启动时报告已注册
  地理围栏的初始状态。
- <PlatformTag platform="ios" /> 可同时监测的 `regions` 数量上限为 [20 个](https://developer.apple.com/documentation/corelocation/monitoring_the_user_s_proximity_to_geographic_regions)。

### 后台权限

要在后台使用位置追踪或地理围栏，你必须请求相应的权限：

- 在 Android 上，你必须同时请求前台和后台权限。
- 在 iOS 上，必须使用 [`requestBackgroundPermissionsAsync`](#locationrequestbackgroundpermissionsasync) 并通过 `Always` 选项授予。

<Collapsible summary="Expo 和 iOS 权限">
iOS 权限分为 `When In Use` 和 `Always` 两类，并分别映射到 Expo 请求的前台和后台位置权限：

- [`requestForegroundPermissionsAsync`](#locationrequestforegroundpermissionsasync) 映射到 `When In Use`
- [`requestBackgroundPermissionsAsync`](#locationrequestbackgroundpermissionsasync) 映射到 `Always`

> **注意：** 当请求 `When In Use` 授权时，用户可以在系统权限对话框中选择 `Allow Once` 来授予**临时访问**。此授权**仅在当前应用会话期间有效**，并且会在应用关闭时自动撤销。

**区分 “Allow Once” 与 “Allow While Using the App”**

遗憾的是，**iOS 无法提供一种方式来检测用户选择的是 `Allow Once` 还是 `Allow While Using the App`**。这两种响应都会导致 `When In Use` 授权。

如果用户选择了 `Allow Once`，而你随后在同一会话中调用 [`requestBackgroundPermissionsAsync`](#locationrequestbackgroundpermissionsasync)，系统**不会再显示另一个提示**。相反，这次请求会**静默失败**，返回的后台权限状态将为**denied**。

**处理 “Allow Once” 场景**

如果你怀疑用户选择了 `Allow Once`，并且需要请求后台权限，他们必须在设置应用中**手动启用后台位置**。你可以使用 `Linking` 在你的应用中打开设置应用：

```js
import { Linking } from 'react-native';

function openSettings() {
  Linking.openURL('app-settings:');
}
```

**逐步权限请求**

可以先请求前台位置访问权限，然后再在之后请求后台位置访问权限。这样可以只在必要时请求权限，从而改善用户体验。

**直接请求后台权限**

如果你在未先请求前台权限的情况下调用 [`requestBackgroundPermissionsAsync`](#locationrequestbackgroundpermissionsasync)，iOS 会将其视为同时请求 `When In Use` 和 `Always` 授权。系统随后会提示用户授予 `When In Use` 访问权限，而当系统判断需要 `Always` 授权时，会显示 `Always` 授权提示。

请记住，用户也可以选择仅授予你的应用 `When In Use` 授权。你必须始终准备好在 `When In Use` 权限下运行。

</Collapsible>

## 延迟位置

在使用后台位置时，你可以配置位置管理器以延迟更新。这有助于通过降低更新频率来节省电量。你可以将更新设置为仅在设备移动了某个距离或经过指定时间间隔后触发。

延迟更新通过 [`LocationTaskOptions`](#locationtaskoptions) 中的 [`deferredUpdatesDistance`](#locationtaskoptions)、[`deferredUpdatesInterval`](#locationtaskoptions) 和 [`deferredTimeout`](#locationtaskoptions) 属性进行配置。

> 延迟位置仅在应用处于后台时适用。

## 用法

如果你正在使用 Android 模拟器或 iOS 模拟器，请确保[已启用位置](#enable-emulator-location)。

<SnackInline label='Location' dependencies={['expo-location', 'expo-constants', 'expo-device']}>

```tsx
import { useState, useEffect } from 'react';
import { Platform, Text, View, StyleSheet } from 'react-native';
/* @hide */
import * as Device from 'expo-device';
/* @end */
import * as Location from 'expo-location';

export default function App() {
  const [location, setLocation] = useState<Location.LocationObject | null>(null);
  const [errorMsg, setErrorMsg] = useState<string | null>(null);

  useEffect(() => {
    async function getCurrentLocation() {
      /* @hide */
      if (Platform.OS === 'android' && !Device.isDevice) {
        setErrorMsg(
          '糟糕，这在 Android 模拟器上的 Snack 中无法运行。请在你的设备上尝试！'
        );
        return;
      }
      /* @end */
      let { status } = await Location.requestForegroundPermissionsAsync();
      if (status !== 'granted') {
        setErrorMsg('访问位置的权限被拒绝');
        return;
      }

      let location = await Location.getCurrentPositionAsync({});
      setLocation(location);
    }

    getCurrentLocation();
  }, []);

  let text = '等待中...';
  if (errorMsg) {
    text = errorMsg;
  } else if (location) {
    text = JSON.stringify(location);
  }

  return (
    <View style={styles.container}>
      <Text style={styles.paragraph}>{text}</Text>
    </View>
  );
}

const styles = StyleSheet.create({
  container: {
    flex: 1,
    alignItems: 'center',
    justifyContent: 'center',
    padding: 20,
  },
  paragraph: {
    fontSize: 18,
    textAlign: 'center',
  },
});
```

</SnackInline>

## 启用模拟器位置

### Android 模拟器

打开 Android Studio，并启动 Android 模拟器。在其中，进入 **Settings** > **Location** 并启用 **Use location**。

<ContentSpotlight
  alt="Location settings in Android Emulator for versions 12 and higher"
  src="/static/images/sdk/location/enable-android-emulator-location.png"
  className="max-w-[360px]"
/>

如果你在模拟器中没有收到位置，你可能需要关闭 **Improve Location Accuracy** 设置。这将关闭 Wi-Fi 定位并且只使用 GPS。然后你可以通过模拟器使用 GPS 数据来操控位置。

对于 Android 12 及更高版本，进入 **Settings** > **Location** > **Location Services** > **Google Location Accuracy**，并关闭 **Improve Location Accuracy**。对于 Android 11 及更低版本，进入 **Settings** > **Location** > **Advanced** > **Google Location Accuracy**，并关闭 **Google Location Accuracy**。

### iOS 模拟器

打开模拟器后，进入 **Features** > **Location**，并选择除 **None** 之外的任意选项。

## 权限

### Android

> **警告** Expo Go for Android 中不可用前台和后台服务。因此，我们建议使用[开发构建](/develop/development-builds/introduction/)以避免限制。

当你安装 `expo-location` 模块时，它会自动添加以下权限：

- `ACCESS_COARSE_LOCATION`：用于大致设备位置
- `ACCESS_FINE_LOCATION`：用于精确设备位置

以下权限是可选的：

- `FOREGROUND_SERVICE` 和 `FOREGROUND_SERVICE_LOCATION`：用于在应用打开但处于后台时访问位置。`FOREGROUND_SERVICE_LOCATION` 仅从 Android 14 开始需要。当你在新构建中启用此项时，你需要[提交应用进行审核并申请使用前台服务权限](https://support.google.com/googleplay/android-developer/answer/13392821?hl=en)。
- `ACCESS_BACKGROUND_LOCATION`：用于在应用处于后台或关闭时访问位置。当你在新构建中启用此项时，你需要[提交应用进行审核并申请使用后台位置权限](https://support.google.com/googleplay/android-developer/answer/9799150?hl=en)。

<AndroidPermissions

#### 排除权限

> **注意**：从应用中的模块排除一个**必需权限**可能会破坏该权限对应的功能。请始终确保包含模块所依赖的所有权限。

当你的 Expo 项目不需要包含某个特定权限时，你可以省略它。例如，如果你的应用不需要访问精确位置，你可以排除 `ACCESS_FINE_LOCATION` 权限。

还可以使用[可用的位置精度](#accuracy)来说明另一个例子。Android 将大致位置精度估计为约 3 平方公里，将精确位置精度估计为约 50 米。例如，如果位置精度值为[Low](#low)，你可以排除 `ACCESS_FINE_LOCATION` 权限。要了解更多关于位置精度级别的信息，请参阅[Android 文档](https://developer.android.com/training/location/permissions#accuracy)。

要了解更多如何排除权限，请参阅[排除 Android 权限](/guides/permissions/#excluding-android-permissions)。
