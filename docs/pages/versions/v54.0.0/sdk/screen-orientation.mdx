屏幕方向是指图形在设备上绘制时所采用的方向。例如，下图中的设备在物理上处于竖直和水平方向，但屏幕方向是纵向。关于物理设备方向，请参阅 [设备运动](devicemotion.mdx) 中的方向部分。

<ContentSpotlight
  alt="不同物理方向下的纵向模式。"
  src="/static/images/screen-orientation-portrait.jpg"
/>

在 Android 和 iOS 平台上，更改屏幕方向会覆盖任何系统设置或用户偏好。在 Android 上，可以在考虑用户首选方向的情况下更改屏幕方向。在 iOS 上，应用无法访问用户和系统设置，且任何对屏幕方向的更改都会覆盖现有设置。

> Web [支持有限](https://caniuse.com/#feat=deviceorientation)。

### 警告

Apple 在 iOS 9 中为 iPad 添加了对 _分屏视图_ 模式的支持。这改变了系统处理屏幕方向的方式。简而言之，对于 iOS，除非你将两个应用并排打开，否则你的 iPad 始终处于横屏模式。若要使用此模块锁定屏幕方向，你需要禁用对此功能的支持。有关 _分屏视图_ 模式的更多信息，请查看 [Apple 官方文档](https://support.apple.com/en-us/HT207582)。

## 在应用配置中进行配置

如果你在项目中使用配置插件（[Continuous Native Generation (CNG)](/workflow/continuous-native-generation/)），你可以使用 `expo-screen-orientation` 内置的 [config plugin](/config-plugins/introduction/) 来进行配置。该插件允许你配置各种无法在运行时设置的属性，并且需要构建新的应用二进制文件后才会生效。如果你的应用**不**使用 CNG，那么你需要手动配置该库。

<ConfigPluginExample>

```json app.json
{
  "expo": {
    "ios": {
      "requireFullScreen": true
    },
    "plugins": [
      [
        "expo-screen-orientation",
        {
          "initialOrientation": "DEFAULT"
        }
      ]
    ]
  }
}
```

</ConfigPluginExample>

<ConfigPluginProperties
  properties={[
    {
      name: 'initialOrientation',
      platform: 'ios',
      description:
        '设置 iOS 的初始屏幕方向。可能的值：`DEFAULT`、`ALL`、`PORTRAIT`、`PORTRAIT_UP`、`PORTRAIT_DOWN`、`LANDSCAPE`、`LANDSCAPE_LEFT`、`LANDSCAPE_RIGHT`',
      default: 'undefined',
    },
  ]}
/>

<ConfigReactNative>

1. 在 Xcode 中使用 `xed ios` 打开 **ios** 目录。如果你没有该目录，请运行 `npx expo prebuild -p ios` 生成一个。
2. 在 Xcode 中勾选 `Requires Full Screen` 复选框。它应位于 **Project Target** > **General** > **Deployment Info** 下。

</ConfigReactNative>
