## app 配置中的配置

如果你的项目使用配置插件（[Continuous Native Generation (CNG)](/workflow/continuous-native-generation/)），你可以使用 `expo-document-picker` 内置的 [config plugin](/config-plugins/introduction/) 进行配置。该插件允许你配置运行时无法设置、且需要构建新的应用二进制文件才会生效的各种属性。如果你的应用**不**使用 CNG，那么你需要手动配置该库。

<ConfigPluginExample>

如果你想启用 [iCloud 存储功能][icloud-entitlement]，请在 [app config](/workflow/configuration/) 文件中将 `expo.ios.usesIcloudStorage` 键设置为 `true`，具体请参见 [configuration properties](../config/app/#usesicloudstorage)。

在本地运行 [EAS Build](/build/introduction) 时，会使用 [iOS capabilities signing](/build-reference/ios-capabilities) 在构建前启用所需的能力。

```json app.json
{
  "expo": {
    "plugins": [
      [
        "expo-document-picker",
        {
          "iCloudContainerEnvironment": "Production"
        }
      ]
    ]
  }
}
```

<ConfigReactNative>

不使用 [EAS Build](/build/introduction) 且希望使用 [iCloud 存储功能][icloud-entitlement] 的应用，必须为其 bundle identifier [手动配置](/build-reference/ios-capabilities#manual-setup) [**带有 CloudKit 支持的 iCloud 服务**](https://developer.apple.com/documentation/bundleresources/entitlements/com_apple_developer_icloud-container-environment)。

如果你通过 [Apple Developer Console](/build-reference/ios-capabilities#apple-developer-console) 启用了 **iCloud** 能力，那么请务必在 `ios/[app]/[app].entitlements` 文件中添加以下 entitlements（其中 `dev.expo.my-app` 为你的 bundle identifier）：

```xml
<key>com.apple.developer.icloud-container-identifiers</key>
<array>
    <string>iCloud.dev.expo.my-app</string>
</array>
<key>com.apple.developer.icloud-services</key>
<array>
    <string>CloudDocuments</string>
</array>
<key>com.apple.developer.ubiquity-container-identifiers</key>
<array>
    <string>iCloud.dev.expo.my-app</string>
</array>
<key>com.apple.developer.ubiquity-kvstore-identifier</key>
<string>$(TeamIdentifierPrefix)dev.expo.my-app</string>
```

Apple Developer Console 还要求创建一个 **iCloud Container**。注册新的容器时，你需要提供该容器的描述和标识符。描述可以填写任意名称。标识符处，添加 `iCloud.<your_bundle_identifier>`（其值与 `com.apple.developer.icloud-container-identifiers` 和 `com.apple.developer.ubiquity-container-identifiers` entitlements 中使用的值相同）。

</ConfigReactNative>

## 与 `expo-file-system` 一起使用

当将 `expo-document-picker` 与 [`expo-file-system`](./filesystem/) 一起使用时，文件系统并不总能在 `expo-document-picker` 选中文件后立即读取该文件。

为了让 `expo-file-system` 在文件被选中后立即读取它，你需要确保将 [`copyToCacheDirectory`](#documentpickeroptions) 选项设置为 `true`。
