`expo-splash-screen` 库中的 `SplashScreen` 模块提供了对原生启动屏行为的控制。默认情况下，当你的应用准备就绪时，启动屏会自动隐藏，但你也可以在更高级的场景中手动控制其可见性。

> 从 **SDK 52** 开始，由于支持最新 Android 启动屏 API 的变化，Expo Go 和开发构建无法完整复现你的 [独立应用](/more/glossary-of-terms/#standalone-app) 中用户会看到的启动屏体验。Expo Go 会显示你的应用图标而不是启动屏，开发构建中的启动屏也不会反映配置插件中设置的所有属性。**强烈建议你在发布构建上测试启动屏，以确保其外观符合预期。**

另请参阅[创建启动屏图片](/develop/user-interface/splash-screen-and-app-icon/#splash-screen)指南，或者[使用浏览器快速生成图标和启动屏](https://buildicon.netlify.app/)。

## 用法

对于大多数应用，你不需要对启动屏做任何特殊处理。它会在应用准备就绪时自动隐藏。你也可以选择配置动画选项：

<Tabs>

<Tab label="使用 Expo Router">

```tsx app/_layout.tsx
import { Stack } from 'expo-router';
import * as SplashScreen from 'expo-splash-screen';

// 设置动画选项。这是可选的。
SplashScreen.setOptions({
  duration: 1000,
  fade: true,
});

export default function RootLayout() {
  return <Stack />;
}
```

</Tab>

<Tab label="不使用 Expo Router">

```tsx App.tsx
import { Text, View } from 'react-native';
import Entypo from '@expo/vector-icons/Entypo';
import * as SplashScreen from 'expo-splash-screen';

// 设置动画选项。这是可选的。
SplashScreen.setOptions({
  duration: 1000,
  fade: true,
});

export default function App() {
  return (
    <View style={{ flex: 1, alignItems: 'center', justifyContent: 'center' }}>
      <Text>启动屏演示！👋</Text>
      <Entypo name="rocket" size={30} />
    </View>
  );
}
```

### 延迟隐藏启动屏

在某些情况下，可能需要延迟隐藏启动屏，直到某些资源加载完成。例如，如果你需要在显示应用内容之前加载 API 数据，可以使用 `preventAutoHideAsync()` 来手动控制启动屏何时隐藏。目标应该是尽快隐藏启动屏。

<Tabs>

<Tab label="使用 Expo Router">

```tsx app/_layout.tsx
import { Stack } from 'expo-router';
import * as SplashScreen from 'expo-splash-screen';
import { useEffect, useState } from 'react';

// 在我们获取资源时保持启动屏可见
SplashScreen.preventAutoHideAsync();

export default function RootLayout() {
  const [isReady, setIsReady] = useState(false);

  useEffect(() => {
    async function doAsyncStuff() {
      try {
        // 在这里执行一些异步操作
      } catch (e) {
        console.warn(e);
      } finally {
        setIsReady(true);
      }
    }

    doAsyncStuff();
  }, []);

  useEffect(() => {
    if (isReady) {
      SplashScreen.hide();
    }
  }, [isReady]);

  if (!isReady) {
    return null;
  }

  return <Stack />;
}
```

</Tab>

## 配置

如果你在项目中使用配置插件（[Continuous Native Generation (CNG)](/workflow/continuous-native-generation/)），可以使用 `expo-splash-screen` 内置的 [配置插件](/config-plugins/introduction/) 来配置它。该插件允许你配置一些无法在运行时设置、且需要构建新的应用二进制文件才会生效的属性。如果你的应用**不**使用 CNG，那么你需要手动配置该库。

**如下所示，使用配置插件是配置启动屏的推荐方式。** 其他方法现在都被视为旧方法，并将在未来被移除。

<ConfigPluginExample>

```json app.json
          },
          "imageWidth": 200

      ]
    ],
  }
}
```

### 启动屏动画

`SplashScreen` 提供了开箱即用的淡入淡出动画。可以使用 `setOptions` 方法进行配置。

```tsx
SplashScreen.setOptions({
  duration: 1000,
  fade: true,
});
```

如果你更想使用自定义动画，请查看 [`with-splash-screen`](https://github.com/expo/examples/tree/master/with-splash-screen) 示例，了解如何为启动屏应用任意动画。你可以通过运行 `npx create-expo-app --example with-splash-screen`，基于此示例初始化一个新项目。
