`react-native-maps` 提供了一个 Map 组件，在 Android 上使用 Google Maps，在 iOS 上使用 Apple Maps 或 Google Maps。

使用 Expo Go 测试项目时不需要额外设置。不过，**若要将应用二进制文件部署到应用商店**，则 Google Maps 需要额外步骤。有关更多信息，请参阅[下面的说明](#deploy-app-with-google-maps)。

## 使用

完整文档请参见 [`react-native-maps/react-native-maps`](https://github.com/react-native-maps/react-native-maps)。

<SnackInline label='MapView' dependencies={['react-native-maps']}>

```jsx
import React from 'react';
import MapView from 'react-native-maps';
import { StyleSheet, View } from 'react-native';

export default function App() {
  return (
    <View style={styles.container}>
      <MapView style={styles.map} />
    </View>
  );
}

const styles = StyleSheet.create({
  container: {
    flex: 1,
  },
  map: {
    width: '100%',
    height: '100%',
  },
});
```

## 使用 Google Maps 部署应用

### Android

> 如果你已经在 Android 上为其他 Google 服务注册过项目，例如 Google Sign In，那么你只需在你的项目中启用 **Maps SDK for Android**，然后直接跳到第 4 步。

<Step label="1">
#### 注册 Google Cloud API 项目并启用 Maps SDK for Android

- 在浏览器中打开 [Google API Manager](https://console.developers.google.com/apis) 并创建一个项目。
- 创建完成后，进入该项目并启用 **Maps SDK for Android**。

</Step>

<Step label="2">
#### 复制你应用的 SHA-1 证书指纹

<Tabs>
<Tab label="For Google Play Store">

- **如果你要将应用部署到 Google Play Store**，你需要至少先[将应用二进制文件上传到 Google Play 控制台](/submit/android/)。这是 Google 生成应用签名凭据所必需的。
- 前往 **[Google Play Console](https://play.google.com/console) > (your app) > Release > Setup > App integrity > App Signing**。
- 复制 **SHA-1 certificate fingerprint** 的值。

</Tab>

<Tab label="For development builds">

- 如果你已经创建了一个[开发构建](/develop/development-builds/introduction/)，你的项目将使用 debug keystore 进行签名。
- 构建完成后，前往你的[项目仪表盘](https://expo.dev/accounts/[username]/projects/[project-name])，然后在 **Configure** 下点击 **Credentials**。
- 在 **Application Identifiers** 下，点击你项目的包名，然后在 **Android Keystore** 下复制 **SHA-1 Certificate Fingerprint** 的值。

</Tab>

</Tabs>

</Step>

<Step label="3">
#### 创建 API key

- 前往 [Google Cloud Credential manager](https://console.cloud.google.com/apis/credentials)，点击 **Create Credentials**，然后选择 **API Key**。
- 在弹窗中，点击 **Edit API key**。
- 在 **Key restrictions** > **Application restrictions** 下，选择 **Android apps**。
- 在 **Restrict usage to your Android apps** 下，点击 **Add an item**。
- 将 **app.json** 中的 `android.package`（例如：`com.company.myapp`）添加到 package name 字段。
- 然后，添加第 2 步中的 **SHA-1 certificate fingerprint** 的值。
- 点击 **Done**，然后点击 **Save**。

</Step>

<Step label="4">
#### 将 API key 添加到你的项目中

- 将你的 **API Key** 复制到项目中的一个 **.env** 文件中，然后将其添加到 **app.json** 的 `android.config.googleMaps.apiKey` 字段中，或者直接复制为：

  ```json
  {
    "android": {
      "config": {
        "googleMaps": {
          "apiKey": "process.env.GOOGLE_MAPS_API_KEY"
        }
      }
    }
  }
  ```

- 在代码中，从 `react-native-maps` 导入 `{ PROVIDER_GOOGLE }`，并将 `provider={PROVIDER_GOOGLE}` 属性添加到你的 `<MapView>` 中。该属性在 Android 和 iOS 上都可用。
- 重新构建应用二进制文件（如果你的应用已经上传，则需要重新提交到 Google Play Store）。验证配置是否成功的一个简单方法是进行一次[模拟器构建](/develop/development-builds/create-a-build/#create-a-development-build-for-emulatorsimulator)。

</Step>

### iOS

> 如果你已经在 iOS 上为其他 Google 服务注册过项目，例如 Google Sign In，那么你只需在你的项目中启用 **Maps SDK for iOS**，然后直接跳到第 3 步。

<Step label="1">
#### 注册 Google Cloud API 项目并启用 Maps SDK for iOS

- 在浏览器中打开 [Google API Manager](https://console.developers.google.com/apis) 并创建一个项目。
- 然后，进入该项目，点击 **Enable APIs and Services**，并启用 **Maps SDK for iOS**。

</Step>

<Step label="2">
#### 创建 API key

- 前往 [Google Cloud Credential manager](https://console.cloud.google.com/apis/credentials)，点击 **Create Credentials**，然后选择 **API Key**。
- 在弹窗中，点击 **Edit API key**。
- 在 **Key restrictions** > **Application restrictions** 下，选择 **iOS apps**。
- 在 **Accept requests from an iOS application with one of these bundle identifiers** 下，点击 **Add an item** 按钮。
- 将 **app.json** 中的 `ios.bundleIdentifier`（例如：`com.company.myapp`）添加到 bundle ID 字段。
- 点击 **Done**，然后点击 **Save**。

</Step>

<Step label="3">
#### 将 API key 添加到你的项目中

- 将你的 **API Key** 复制到项目中的一个 **.env** 文件中，然后将其添加到 **app.json** 的 `ios.config.googleMapsApiKey` 字段中，或者直接复制为：

  ```json
  {
    "ios": {
      "config": {
        "googleMapsApiKey": "process.env.GOOGLE_MAPS_API_KEY"
      }
    }
  }
  ```

- 在代码中，从 `react-native-maps` 导入 `{ PROVIDER_GOOGLE }`，并将 `provider={PROVIDER_GOOGLE}` 属性添加到你的 `<MapView>` 中。该属性在 Android 和 iOS 上都可用。
- 重新构建应用二进制文件。验证配置是否成功的一个简单方法是进行一次[模拟器构建](/develop/development-builds/create-a-build/#create-a-development-build-for-emulatorsimulator)。

</Step>
