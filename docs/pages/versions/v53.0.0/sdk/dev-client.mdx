`expo-dev-client` 会为你的调试构建添加各种有用的开发工具：

- 可配置的启动器 UI，因此你可以启动更新（例如来自 [PR 预览](/develop/development-builds/development-workflows#pr-previews)）并在开发服务器之间切换，而无需重新编译原生应用
- 改进的调试工具（例如支持 [检查网络请求](/debugging/tools/#inspecting-network-requests)）
- [强大且可扩展的开发者菜单 UI](/debugging/tools#developer-menu)

Expo 文档将包含 `expo-dev-client` 的调试构建称为 [development builds](/develop/development-builds/introduction/)。

## 安装

<APIInstallSection hideBareInstructions />

如果你要在一个[现有的 React Native 应用](/bare/overview/)中安装它，请先在你的项目中安装 [`expo`](/bare/installing-expo-modules/)。然后，按照[在现有 React Native 项目中安装 `expo-dev-client`](/bare/install-dev-builds-in-bare/)中的说明进行操作。

## app 配置中的配置

如果你在项目中使用 config plugins（[Continuous Native Generation (CNG)](/workflow/continuous-native-generation/)），你可以使用其内置的 [config plugin](/config-plugins/introduction/) 来配置 development client launcher。该插件允许你配置各种无法在运行时设置、并且需要构建新的应用二进制文件后才会生效的属性。如果你的应用**不**使用 CNG，那么你需要手动配置该库。

<ConfigPluginExample>

```json app.json
{
  "expo": {
    "plugins": [
      [
        "expo-dev-client",
        {
          "launchMode": "most-recent"
        }
      ]
    ]
  }
}
```

</ConfigPluginExample>
