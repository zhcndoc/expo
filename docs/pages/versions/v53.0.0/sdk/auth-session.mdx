`AuthSession` 通过利用 [WebBrowser](webbrowser.mdx) 和 [Crypto](crypto.mdx)，为你的应用启用基于网页浏览器的身份验证（例如，基于浏览器的 OAuth 流程）。有关实现细节，请参考此文档；有关使用方法，请参阅 [Authentication](/guides/authentication/) 指南。

> **注意：** `AuthSession` 支持通用的基于浏览器的 OAuth 和 OpenID Connect 身份验证工作流。若有可用的方案，我们建议使用身份提供方提供的库，因为它会处理该提供方特有的实现细节。例如，Google 身份验证请使用 [`@react-native-google-signin/google-signin`](/guides/google-authentication/)，Facebook 身份验证请使用 [`react-native-fbsdk-next`](/guides/facebook-authentication/)。更多信息请参阅 [Authentication](/develop/authentication/) 概览。

### 在独立应用中的使用

```json app.json
{
  "expo": {
    "scheme": "mycoolredirect"
  }
}
```

为了能够通过深度链接回到你的应用，你需要在项目的 app 配置中设置一个 `scheme`，然后构建你的独立应用（它不能通过更新来修改）。如果你不包含 scheme，身份验证流程会完成，但它无法将信息返回到你的应用中，用户将不得不手动退出身份验证弹窗（从而产生一个取消事件）。

## 基于网页浏览器的身份验证流程如何工作

移动应用中基于浏览器的身份验证通常流程如下：

- **开始**：用户点击登录按钮
- **打开网页浏览器**：应用打开网页浏览器并跳转到身份提供方的登录页面。打开的登录页 URL 通常包含用于识别应用的信息，以及在成功后重定向的 URL。_注意：网页浏览器应与系统网页浏览器共享 cookie，这样如果用户已经在系统浏览器中完成登录，就无需再次登录——Expo 的 [WebBrowser](webbrowser.mdx) API 会负责处理这一点。_
- **身份提供方重定向**：身份验证成功后，身份提供方应通过重定向到应用在登录页面查询参数中提供的 URL，将用户带回应用（[了解更多移动应用中的链接工作原理](/linking/overview/)），_前提是该 URL 位于允许的重定向 URL 白名单中_。对重定向 URL 进行白名单控制很重要，可以防止恶意行为者冒充你的应用。重定向会在 URL 中包含数据（例如用户 id 和 token），这些数据可能位于 location hash、查询参数中，或两者都有。
- **应用处理重定向**：由应用处理该重定向，并从重定向 URL 中解析数据。

## 安全注意事项

- **切勿将任何密钥放入你的应用代码中，这样做没有安全的方法！**相反，你应该将密钥存储在服务器上，并提供一个端点，为客户端发起 API 调用并将数据返回。

## 高级用法

### 在 Linking 处理器中过滤 AuthSession 事件

你可能希望处理进入应用的链接，有很多原因，例如推送通知或普通的深度链接（你可以在 [Linking](/linking/overview/) 中了解更多）；身份验证重定向只是深度链接的一种，而 `AuthSession` 会为你处理这些特定链接。在你自己的 `Linking.addEventListener` 处理器中，可以通过检查 URL 是否包含 `+expo-auth-session` 字符串来过滤掉由 `AuthSession` 处理的深度链接——如果包含，就可以忽略它。之所以这样可行，是因为 `AuthSession` 会在默认的 `returnUrl` 中添加 `+expo-auth-session`；不过，如果你提供了自己的 `returnUrl`，你可能需要考虑添加一个类似的标识符，以便从其他处理器中过滤掉 `AuthSession` 事件。

### 与 React Navigation 一起使用

如果你正在使用 React Navigation 进行深度链接，那么通过 `Linking.addEventListener` 进行过滤并不够，因为深度链接的处理方式不同（[详情](https://reactnavigation.org/docs/configuring-links/#advanced-cases)）。相反，为了过滤这些事件，请在你的 linking 配置中添加一个自定义的 `getStateFromPath` 函数，然后像上面所述那样按 URL 进行过滤。
