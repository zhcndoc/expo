`expo-sensors` 中的 `DeviceMotion` 提供对设备运动和方向传感器的访问。所有数据都以穿过设备的三个轴来表示。根据竖屏方向：X 轴从左到右，Y 轴从下到上，Z 轴垂直于屏幕，从后到前。

## 在应用配置中进行配置

如果你在项目中使用配置插件（[Continuous Native Generation (CNG)](/workflow/continuous-native-generation/)），你可以使用其内置的 [config plugin](/config-plugins/introduction/) 来配置来自 `expo-sensor` 的 `DeviceMotion`。该插件允许你配置各种无法在运行时设置、且需要构建新的应用二进制文件后才会生效的属性。如果你的应用**不**使用 CNG，那么你需要手动配置该库。

<ConfigPluginExample>

```json app.json
{
  "expo": {
    "plugins": [
      [
        "expo-sensors",
        {
          "motionPermission": "Allow $(PRODUCT_NAME) to access your device motion."
        }
      ]
    ]
  }
}
```

</ConfigPluginExample>

<ConfigPluginProperties
  properties={[
    {
      name: 'motionPermission',
      platform: 'ios',
      description: '用于设置 [`NSMotionUsageDescription`](#ios) 权限消息的字符串。',
      default: '"Allow $(PRODUCT_NAME) to access your device motion"',
    },
  ]}
/>

<ConfigReactNative>

如果你没有使用 Continuous Native Generation ([CNG](/workflow/continuous-native-generation/))，或者你正在手动使用原生 **ios** 项目，那么你需要在原生项目中配置 `NSMotionUsageDescription` 键，以访问 `DeviceMotion` 统计信息：

```xml ios/[app]/Info.plist
<key>NSMotionUsageDescription</key>
<string>Allow $(PRODUCT_NAME) to access your device motion</string>
```

</ConfigReactNative>
