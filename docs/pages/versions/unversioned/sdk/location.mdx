`expo-location` 允许从设备读取地理位置信息。你的应用可以轮询当前位置信息，或者订阅位置更新事件。

## 在 app 配置中进行配置

如果你的项目使用 config plugins（[Continuous Native Generation (CNG)](/workflow/continuous-native-generation/)），你可以使用 `expo-location` 内置的 [config plugin](/config-plugins/introduction/) 来进行配置。该插件允许你配置一些无法在运行时设置、且需要构建新的应用二进制文件才会生效的属性。如果你的应用**不**使用 CNG，那么你需要手动配置该库。

<ConfigPluginExample>

```json app.json
{
  "expo": {
    "plugins": [
      [
        "expo-location",
        {
          "locationAlwaysAndWhenInUsePermission": "允许 $(PRODUCT_NAME) 使用你的位置信息。"
        }
      ]
    ]
  }
}
```

### 后台位置

后台位置允许你的应用在后台运行时接收位置更新，并且包括通过地理围栏进行的位置更新和区域监测。此功能受平台 API 限制和系统约束影响：

- 当用户终止应用时，后台位置会停止。
- 当用户重新启动应用时，后台位置会恢复。
- <PlatformTag platform="android" /> 由于平台限制，终止的应用在发生位置或地理围栏事件时不会自动重启。
- <PlatformTag platform="ios" /> 当新的地理围栏事件发生时，系统会重启已终止的应用。

> **信息** 在 Android 上，将应用从最近任务列表中移除的结果因设备厂商而异。例如，有些实现会将其视为杀死应用。更多关于这些差异的信息请见：[https://dontkillmyapp.com](https://dontkillmyapp.com)。

### 后台位置配置&ensp;<PlatformTag platform="ios" />

要在 iOS 上运行后台位置，你需要在应用的 **Info.plist** 文件中的 `UIBackgroundModes` 数组里添加 `location` 值。

**如果你使用的是 [CNG](/workflow/continuous-native-generation/)**，所需的 `UIBackgroundModes` 配置会在 prebuild 过程中自动应用。

<Collapsible summary="在 iOS 上手动配置 UIBackgroundModes">

如果你没有使用 Continuous Native Generation ([CNG](/workflow/continuous-native-generation/))，或者你正在使用原生 iOS 项目，那么你需要将以下内容添加到 **Expo.plist** 文件中：

```xml ios/project-name/Supporting/Expo.plist
<key>UIBackgroundModes</key>
  <array>
    <string>location</string>
  </array>
```

</Collapsible>

### 后台位置方法

要使用后台位置方法，需要满足以下要求：

- 必须授予位置权限。
- 必须在顶层作用域中使用 [`TaskManager.defineTask`](task-manager.mdx#taskmanagerdefinetasktaskname-taskexecutor) 定义后台位置任务。
- <PlatformTag platform="ios" className="float-left" /> 必须在 **Info.plist** 文件中指定 `"location"` 后台模式。请参阅[后台位置配置](#background-location-configuration)。
- <PlatformTag platform="ios" className="float-left" /> 由于 Expo Go 应用不支持后台位置，因此你必须使用[开发构建](/develop/development-builds/introduction/)来使用后台位置。

### 地理围栏方法

要使用地理围栏方法，需要满足以下要求：

- 必须授予位置权限。
- 必须在顶层作用域中使用 [`TaskManager.defineTask`](task-manager.mdx#taskmanagerdefinetasktaskname-taskexecutor) 定义地理围栏任务。

使用地理围栏时，适用以下平台差异：

- <PlatformTag platform="android" /> 每个应用最多允许[100 个](https://developer.android.com/develop/sensors-and-location/location/geofencing)活动地理围栏。
- <PlatformTag platform="ios" /> Expo Location 会在应用启动时报告已注册地理围栏的初始状态。
- <PlatformTag platform="ios" /> 可同时监测的 `regions` 数量[上限为 20](https://developer.apple.com/documentation/corelocation/monitoring_the_user_s_proximity_to_geographic_regions)。

### 后台权限

要在后台使用位置跟踪或地理围栏，你必须请求相应的权限：

- 在 Android 上，你必须同时请求前台和后台权限。
- 在 iOS 上，必须使用 [`requestBackgroundPermissionsAsync`](#locationrequestbackgroundpermissionsasync) 以 `Always` 选项授予权限。

<Collapsible summary="Expo 和 iOS 权限">
iOS 权限分为 `When In Use` 和 `Always` 两类，并映射到 Expo 请求的前台和后台位置权限：

- [`requestForegroundPermissionsAsync`](#locationrequestforegroundpermissionsasync) 对应 `When In Use`
- [`requestBackgroundPermissionsAsync`](#locationrequestbackgroundpermissionsasync) 对应 `Always`

> **注意：** 在请求 `When In Use` 授权时，用户可以在系统权限对话框中选择 `Allow Once` 来授予**临时访问**。此授权将**仅对当前应用会话有效**，并会在应用关闭时自动撤销。

**区分 “Allow Once” 与 “Allow While Using the App”**

遗憾的是，**iOS 无法提供一种方式来检测用户选择的是 `Allow Once` 还是 `Allow While Using the App`**。这两种响应都会得到 `When In Use` 授权。

如果用户选择了 `Allow Once`，而你随后在同一会话中调用 [`requestBackgroundPermissionsAsync`](#locationrequestbackgroundpermissionsasync)，系统**不会再显示另一个提示**。相反，请求会**静默失败**，返回的后台权限状态将是**被拒绝**。

**处理 “Allow Once” 场景**

如果你怀疑用户选择了 `Allow Once` 且需要请求后台权限，他们必须在设置应用中**手动启用后台位置**。你可以使用 `Linking` 在应用中打开设置应用：

```js
import { Linking } from 'react-native';

function openSettings() {
  Linking.openURL('app-settings:');
}
```

**逐步请求权限**

可以先请求前台位置访问权限，然后再在之后请求后台位置访问权限。这样可以仅在必要时请求权限，从而改善用户体验。

**直接请求后台权限**

如果你在未先请求前台权限的情况下调用 [`requestBackgroundPermissionsAsync`](#locationrequestbackgroundpermissionsasync)，iOS 会将其视为同时请求 `When In Use` 和 `Always` 授权。系统随后会提示用户授予 `When In Use` 访问权限，而当系统判断需要 `Always` 授权时，会显示 `Always` 授权提示。

请记住，用户也可以选择仅授予你的应用 `When In Use` 授权。你必须始终准备好在 `When In Use` 权限下运行。

</Collapsible>

## 延迟位置

在使用后台位置时，你可以配置位置管理器以延迟更新。这通过降低更新频率有助于节省电量。你可以将更新设置为仅在设备移动了某个距离后，或在经过指定时间间隔后触发。

延迟更新通过 [`LocationTaskOptions`](#locationtaskoptions) 进行配置，使用 [`deferredUpdatesDistance`](#locationtaskoptions)、[`deferredUpdatesInterval`](#locationtaskoptions) 和 [`deferredTimeout`](#locationtaskoptions) 属性。

> 延迟位置仅在应用处于后台时适用。

## 用法

如果你正在使用 Android 模拟器或 iOS 模拟器，请确保 [已启用位置](#enable-emulator-location)。

<SnackInline label='位置' dependencies={['expo-location', 'expo-constants', 'expo-device']}>

```tsx
import { useState, useEffect } from 'react';
import { Platform, Text, View, StyleSheet } from 'react-native';
/* @hide */
import * as Device from 'expo-device';
/* @end */
import * as Location from 'expo-location';

export default function App() {
  const [location, setLocation] = useState<Location.LocationObject | null>(null);
  const [errorMsg, setErrorMsg] = useState<string | null>(null);

  useEffect(() => {
    async function getCurrentLocation() {
      /* @hide */
      if (Platform.OS === 'android' && !Device.isDevice) {
        setErrorMsg(
          '糟糕，这在 Android 模拟器中的 Snack 上无法运行。请在你的设备上尝试！'
        );
        return;
      }
      /* @end */
      let { status } = await Location.requestForegroundPermissionsAsync();
      if (status !== 'granted') {
        setErrorMsg('访问位置的权限被拒绝');
        return;
      }

      let location = await Location.getCurrentPositionAsync({});
      setLocation(location);
    }

    getCurrentLocation();
  }, []);

  let text = '等待中...';
  if (errorMsg) {
    text = errorMsg;
  } else if (location) {
    text = JSON.stringify(location);
  }

  return (
    <View style={styles.container}>
      <Text style={styles.paragraph}>{text}</Text>
    </View>
  );
}

const styles = StyleSheet.create({
  container: {
    flex: 1,
    alignItems: 'center',
    justifyContent: 'center',
    padding: 20,
  },
  paragraph: {
    fontSize: 18,
    textAlign: 'center',
  },
});
```

</SnackInline>

## 启用模拟器位置

### Android 模拟器

打开 Android Studio，并启动 Android 模拟器。在其中，进入 **设置** > **位置**，并启用 **使用位置**。

<ContentSpotlight
  alt="Android 模拟器中 12 及更高版本的位置设置"
  src="/static/images/sdk/location/enable-android-emulator-location.png"
  className="max-w-[360px]"
/>

如果你没有在模拟器中接收到位置，你可能需要关闭 **提高位置精度** 设置。这将关闭 Wi-Fi 定位并且仅使用 GPS。然后你可以通过模拟器使用 GPS 数据来操控位置。

对于 Android 12 及更高版本，进入 **设置** > **位置** > **位置服务** > **Google 位置精度**，并关闭 **提高位置精度**。对于 Android 11 及更低版本，进入 **设置** > **位置** > **高级** > **Google 位置精度**，并关闭 **Google 位置精度**。

### iOS 模拟器

在模拟器打开的情况下，进入 **功能** > **位置**，并选择除 **无** 之外的任意选项。

## 权限

### Android

> **警告** Expo Go for Android 中不提供前台和后台服务。作为替代，我们建议使用[开发构建](/develop/development-builds/introduction/)以避免限制。

当你安装 `expo-location` 模块时，它会自动添加以下权限：

- `ACCESS_COARSE_LOCATION`：用于近似设备位置
- `ACCESS_FINE_LOCATION`：用于精确设备位置

以下权限是可选的：

- `FOREGROUND_SERVICE` 和 `FOREGROUND_SERVICE_LOCATION`：用于在应用打开但处于后台时访问位置。`FOREGROUND_SERVICE_LOCATION` 仅从 Android 14 开始需要。当你在新构建中启用此项时，你需要[提交应用进行审核并申请使用前台服务权限](https://support.google.com/googleplay/android-developer/answer/13392821?hl=en)。
- `ACCESS_BACKGROUND_LOCATION`：用于在应用处于后台或关闭时访问位置。当你在新构建中启用此项时，你需要[提交应用进行审核并申请使用后台位置权限](https://support.google.com/googleplay/android-developer/answer/9799150?hl=en)。

#### 排除权限

> **注意**：在你的应用中从某个模块排除一个**必需权限**可能会破坏该权限对应的功能。请始终确保包含模块所依赖的所有权限。

当你的 Expo 项目从包含某些特定权限中无法获益时，你可以省略它。例如，如果你的应用不需要访问精确位置，你可以排除 `ACCESS_FINE_LOCATION` 权限。

另一个示例可以使用[可用的位置精度](#accuracy)来说明。Android 将近似位置精度估计定义为约 3 平方公里，将精确位置精度估计定义为约 50 米。例如，如果位置精度值为 [Low](#low)，你可以排除 `ACCESS_FINE_LOCATION` 权限。要了解有关位置精度级别的更多信息，请参阅[Android 文档](https://developer.android.com/training/location/permissions#accuracy)。

要进一步了解如何排除权限，请参阅[排除 Android 权限](/guides/permissions/#excluding-android-permissions)。
