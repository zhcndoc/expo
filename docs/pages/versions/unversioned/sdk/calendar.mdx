`expo-calendar` 提供了一个用于与设备系统日历、事件、提醒事项以及相关记录交互的 API。

此外，它还提供了用于启动[系统提供的日历 UI](#launching-system-provided-calendar-dialogs)的方法，以便让用户查看或编辑事件。在 Android 上，这些方法会通过 Intent 启动系统日历应用。在 iOS 上，它们会以模态方式呈现 [`EKEventViewController`](https://developer.apple.com/documentation/eventkitui/ekeventviewcontroller) 或 [`EKEventEditViewController`](https://developer.apple.com/documentation/eventkitui/ekeventeditviewcontroller)。

## 在 app 配置中进行配置

如果你在项目中使用配置插件（[Continuous Native Generation (CNG)](/workflow/continuous-native-generation/)），你可以使用 `expo-calendar` 内置的[配置插件](/config-plugins/introduction/)来进行配置。该插件允许你配置一些无法在运行时设置、且需要重新构建新的应用二进制文件才会生效的属性。如果你的应用**不**使用 CNG，那么你需要手动配置该库。

<ConfigPluginExample>

```json app.json
{
  "expo": {
    "plugins": [
      [
        "expo-calendar",
        {
          "calendarPermission": "应用需要访问你的日历。"
        }
      ]
    ]
  }
}
```

<ConfigReactNative>

如果你没有使用 Continuous Native Generation ([CNG](/workflow/continuous-native-generation/))（而是手动使用原生 **android** 和 **ios** 项目），那么你需要在原生项目中配置以下权限：

- 对于 Android，将 `android.permission.READ_CALENDAR` 和 `android.permission.WRITE_CALENDAR` 权限添加到项目的 **android/app/src/main/AndroidManifest.xml** 中：

  ```xml
  <uses-permission android:name="android.permission.READ_CALENDAR" />
  <uses-permission android:name="android.permission.WRITE_CALENDAR" />
  ```

- 对于 iOS，将 `NSCalendarsUsageDescription` 和 `NSRemindersUsageDescription` 添加到项目的 **ios/[app]/Info.plist** 中：

  ```xml
  <key>NSCalendarsUsageDescription</key>
  <string>允许 $(PRODUCT_NAME) 访问你的日历</string>
  <key>NSRemindersUsageDescription</key>
  <string>允许 $(PRODUCT_NAME) 访问你的提醒事项</string>
  ```

## 用法

<SnackInline label='基础日历用法' dependencies={['expo-calendar']}>

```jsx
import { useEffect } from 'react';
import { StyleSheet, View, Text, Button, Platform } from 'react-native';
import * as Calendar from 'expo-calendar';

export default function App() {
  useEffect(() => {
    (async () => {
      const { status } = await Calendar.requestCalendarPermissionsAsync();
      if (status === 'granted') {
        const calendars = await Calendar.getCalendarsAsync(Calendar.EntityTypes.EVENT);
        console.log('这里是你的所有日历：');
        console.log({ calendars });
      }
    })();
  }, []);

  return (
    <View style={styles.container}>
      <Text>日历模块示例</Text>
      <Button title="创建一个新日历" onPress={createCalendar} />
    </View>
  );
}

async function getDefaultCalendarSource() {
  const defaultCalendar = await Calendar.getDefaultCalendarAsync();
  return defaultCalendar.source;
}

async function createCalendar() {
  const defaultCalendarSource =
    Platform.OS === 'ios'
      ? await getDefaultCalendarSource()
      : { isLocalAccount: true, name: 'Expo Calendar' };
  const newCalendarID = await Calendar.createCalendarAsync({
    title: 'Expo Calendar',
    color: 'blue',
    entityType: Calendar.EntityTypes.EVENT,
    sourceId: defaultCalendarSource.id,
    source: defaultCalendarSource,
    name: 'internalCalendarName',
    ownerAccount: 'personal',
    accessLevel: Calendar.CalendarAccessLevel.OWNER,
  });
  console.log(`你的新日历 ID 是：${newCalendarID}`);
}

const styles = StyleSheet.create({
  container: {
    flex: 1,
    backgroundColor: '#fff',
    alignItems: 'center',
    justifyContent: 'space-around',
  },
});
```

## 权限

### Android

如果你只打算使用[系统提供的日历 UI](#launching-system-provided-calendar-dialogs)，则不需要请求任何权限。

否则，你必须将以下权限添加到 **app.json** 中 [`expo.android.permissions`](../config/app/#permissions) 数组里。

<AndroidPermissions permissions={['READ_CALENDAR', 'WRITE_CALENDAR']} />

### iOS

如果你只打算使用系统提供的日历 UI，并通过 [`createEventInCalendarAsync`](#createeventincalendarasynceventdata-presentationoptions) 创建事件，则不需要请求权限。

该库使用以下用途说明键：

<IOSPermissions permissions={['NSCalendarsUsageDescription', 'NSRemindersUsageDescription']} />
