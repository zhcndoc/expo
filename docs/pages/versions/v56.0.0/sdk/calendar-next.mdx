> **info** `next` 版本的 Calendar API 包含在 `expo-calendar` 库中。它可以与之前的 API 一起使用，并提供一种更简化、面向对象的方式来执行日历操作。

> **important** 为了提供更快的更新，`expo-calendar/next` 目前在 Expo Go 和 Snack 中不受支持。要使用它，请创建一个 [开发构建](/develop/development-builds/create-a-build/)。

`expo-calendar` 提供了一个用于与设备系统日历、事件、提醒以及相关记录交互的 API。

此外，它还提供了用于启动[系统提供的日历 UI](#launching-system-provided-calendar-dialogs) 的方法，让用户可以查看或编辑事件。在 iOS 上，它们会以模态形式显示 [`EKEventViewController`](https://developer.apple.com/documentation/eventkitui/ekeventviewcontroller) 或 [`EKEventEditViewController`](https://developer.apple.com/documentation/eventkitui/ekeventeditviewcontroller)。

## 在 app 配置中进行配置

如果你在项目中使用配置插件（[Continuous Native Generation (CNG)](/workflow/continuous-native-generation/)），可以使用 `expo-calendar` 内置的[配置插件](/config-plugins/introduction/)来进行配置。该插件允许你配置一些无法在运行时设置的属性，这些属性需要构建一个新的应用二进制文件后才会生效。如果你的应用**不**使用 CNG，那么你需要手动配置该库。

<ConfigPluginExample>

```json app.json
{
  "expo": {
    "plugins": [
      [
        "expo-calendar",
        {
          "calendarPermission": "该应用需要访问你的日历。"
        }
      ]
    ]
  }
}
```

## 使用

```jsx
import * as Calendar from 'expo-calendar/next';
import { useEffect } from 'react';
import { StyleSheet, View, Text, Button } from 'react-native';

const BasicUsage = () => {
  useEffect(() => {
    (async () => {
      const { status } = await Calendar.requestCalendarPermissions();
      if (status === 'granted') {
        const calendars = Calendar.getCalendars(Calendar.EntityTypes.EVENT);
        console.log('以下是你的所有日历：');
        console.log(JSON.stringify(calendars));
      }
    })();
  }, []);

  return (
    <View style={styles.container}>
      <Text>日历模块示例</Text>
      <Button title="创建一个新日历" onPress={createCalendar} />
    </View>
  );
};

async function createCalendar() {
  const newCalendar = await Calendar.createCalendar({
    title: 'Expo Calendar',
    color: 'blue',
    entityType: Calendar.EntityTypes.EVENT,
  });
  console.log(`你的新日历：${JSON.stringify(newCalendar)}`);
}

const styles = StyleSheet.create({
  container: {
    flex: 1,
    backgroundColor: '#fff',
    alignItems: 'center',
    justifyContent: 'space-around',
  },
});
```

## 权限

### Android

如果你只打算使用[系统提供的日历 UI](#launching-system-provided-calendar-dialogs)，则无需请求任何权限。

否则，你必须将以下权限添加到 **app.json** 中的 [`expo.android.permissions`](../config/app/#permissions) 数组里。

<AndroidPermissions permissions={['READ_CALENDAR', 'WRITE_CALENDAR']} />

### iOS

如果你只打算使用系统提供的日历 UI，通过 [`createEventInCalendarAsync`](#createeventincalendarasynceventdata-presentationoptions) 来创建事件，则无需请求权限。

此库使用以下用途说明键：

<IOSPermissions permissions={['NSCalendarsUsageDescription', 'NSRemindersUsageDescription']} />
