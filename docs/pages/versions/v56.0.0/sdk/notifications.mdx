`expo-notifications` 提供了一个用于获取推送通知令牌，以及展示、调度、接收和响应通知的 API。

> **warning** 从 SDK 53 开始，`expo-notifications` 提供的推送通知（远程通知）功能在 Android 版 Expo Go 中不可用。使用推送通知需要一个 [开发构建](/develop/development-builds/introduction/)。本地通知（应用内通知）仍可在 Expo Go 中使用。

## 功能

- 为特定日期或从现在起的某个时间安排一次性通知
- 按时间间隔重复安排通知（或在 iOS 上按日历日期匹配）
- 获取并设置应用角标图标数字
- 获取原生设备推送令牌，以便你可以使用 FCM（适用于 Android）和 APNs（适用于 iOS）发送推送通知
- 获取 Expo 推送令牌，以便你可以使用 [Expo Push Service](/push-notifications/sending-notifications/) 发送推送通知
- 监听前台和后台传入的通知
- 监听与通知的交互
- 当应用处于前台时处理通知
- 通过通知中心/托盘以命令式方式关闭通知
- 创建、更新和删除 [Android 通知渠道](https://developer.android.com/develop/ui/views/notifications/channels)
- 为 Android 上的通知设置自定义图标和颜色

### 已知问题 <PlatformTags platforms={['android']} />

当从推送通知启动应用时，在 **Android 开发构建** 中，启动画面有约 70% 的概率无法正确显示。图标和淡出动画可能不会按预期出现。

- 图标可能缺失
- 淡出动画可能不会运行
- 只会短暂闪现背景颜色

此问题仅影响调试构建，不会在发布构建中出现。作为解决方法，请在发布模式下测试通知启动（`npx expo run:android --variant release`），以获得准确的行为。

## 用法

查看下面的示例 Snack，了解 Notifications 的实际效果。推送通知可在真机、带有 Google Play services 的 Android 模拟器，以及 Xcode 14 或更高版本（macOS 13+、iOS 16+）上的 iOS 模拟器中使用。

<SnackInline label='Push Notifications' dependencies={['expo-constants', 'expo-notifications']}>

```tsx
import { useState, useEffect } from 'react';
import { Text, View, Button, Platform } from 'react-native';
import * as Notifications from 'expo-notifications';
import Constants from 'expo-constants';

Notifications.setNotificationHandler({
  handleNotification: async () => ({
    shouldPlaySound: false,
    shouldSetBadge: false,
    shouldShowBanner: true,
    shouldShowList: true,
  }),
});

export default function App() {
  const [expoPushToken, setExpoPushToken] = useState('');
  const [channels, setChannels] = useState<Notifications.NotificationChannel[]>([]);
  const [notification, setNotification] = useState<Notifications.Notification | undefined>(
    undefined
  );

  useEffect(() => {
    registerForPushNotificationsAsync().then(token => token && setExpoPushToken(token));

    if (Platform.OS === 'android') {
      Notifications.getNotificationChannelsAsync().then(value => setChannels(value ?? []));
    }
    const notificationListener = Notifications.addNotificationReceivedListener(notification => {
      setNotification(notification);
    });

    const responseListener = Notifications.addNotificationResponseReceivedListener(response => {
      console.log(response);
    });

    return () => {
      notificationListener.remove();
      responseListener.remove();
    };
  }, []);

  return (
    <View
      style={{
        flex: 1,
        alignItems: 'center',
        justifyContent: 'space-around',
      }}>
      <Text>Your expo push token: {expoPushToken}</Text>
      <Text>{`Channels: ${JSON.stringify(
        channels.map(c => c.id),
        null,
        2
      )}`}</Text>
      <View style={{ alignItems: 'center', justifyContent: 'center' }}>
        <Text>Title: {notification && notification.request.content.title} </Text>
        <Text>Body: {notification && notification.request.content.body}</Text>
        <Text>Data: {notification && JSON.stringify(notification.request.content.data)}</Text>
      </View>
      <Button
        title="Press to schedule a notification"
        onPress={async () => {
          await schedulePushNotification();
        }}
      />
    </View>
  );
}

async function schedulePushNotification() {
  await Notifications.scheduleNotificationAsync({
    content: {
      title: "You've got mail! 📬",
      body: 'Here is the notification body',
      data: { data: 'goes here', test: { test1: 'more data' } },
    },
    trigger: {
      type: Notifications.SchedulableTriggerInputTypes.TIME_INTERVAL,
      seconds: 2,
    },
  });
}

async function registerForPushNotificationsAsync() {
  let token;

  if (Platform.OS === 'android') {
    await Notifications.setNotificationChannelAsync('myNotificationChannel', {
      name: '显示权限提示所需的渠道',
      importance: Notifications.AndroidImportance.MAX,
      vibrationPattern: [0, 250, 250, 250],
      lightColor: '#FF231F7C',
    });
  }

  const { status: existingStatus } = await Notifications.getPermissionsAsync();
  let finalStatus = existingStatus;
  if (existingStatus !== 'granted') {
    const { status } = await Notifications.requestPermissionsAsync();
    finalStatus = status;
  }
  if (finalStatus !== 'granted') {
    alert('获取推送通知令牌失败！');
    return;
  }
  // 了解更多关于 projectId 的信息：
  // https://docs.expo.dev/push-notifications/push-notifications-setup/#configure-projectid
  // 此处使用的是 EAS projectId。
  try {
    const projectId =
      Constants?.expoConfig?.extra?.eas?.projectId ?? Constants?.easConfig?.projectId;
    if (!projectId) {
      throw new Error('未找到 Project ID');
    }
    token = (
      await Notifications.getExpoPushTokenAsync({
        projectId,
      })
    ).data;
    console.log(token);
  } catch (e) {
    token = `${e}`;
  }

  return token;
}
```

### 使用导航处理推送通知

如果你希望在收到推送通知时深度链接到应用中的某个特定屏幕，可以配置 Expo 的任一导航系统来实现。

<Tabs>

<Tab label="Expo Router">

你可以使用 Expo Router 的[内置深度链接](/router/basics/core-concepts/#2-all-pages-have-a-url)来处理来自推送通知的传入 URL。只需配置根布局以监听传入和初始通知事件。

```tsx app/_layout.tsx
import { useEffect } from 'react';
import * as Notifications from 'expo-notifications';
import { router } from 'expo-router';

function useNotificationObserver() {
  useEffect(() => {
    function redirect(notification: Notifications.Notification) {
      const url = notification.request.content.data?.url;
      if (typeof url === 'string') {
        /* @info 推送该 URL。你可能希望在导航前验证其格式。 */
        router.push(url);
        /* @end */
      }
    }

    /* @info 处理初始推送通知。 */
    const response = Notifications.getLastNotificationResponse();
    if (response?.notification) {
      redirect(response.notification);
    }

    /* @info 监听运行时通知。 */
    const subscription = Notifications.addNotificationResponseReceivedListener(response => {
      /* @end */
      redirect(response.notification);
    });

    return () => {
      subscription.remove();
    };
  }, []);
}

export default function Layout() {
  /* @info 在根部进行监听。请确保此布局绝不返回 <b>null</b>，否则导航将不会被处理。 */
  useNotificationObserver();
  /* @end */

  return <Slot />;
}
```

</Tab>

<Tab label="React Navigation">

React Navigation 的手动 [linking 配置](https://reactnavigation.org/docs/navigation-container#linking) 可以配置为处理来自推送通知的传入重定向：

```tsx App.tsx
import React from 'react';
import { Linking } from 'react-native';
import * as Notifications from 'expo-notifications';
import { NavigationContainer } from '@react-navigation/native';

export default function App() {
  return (
    <NavigationContainer
      linking={{
        config: {
          // linking 的配置
        },
        async getInitialURL() {
          // 首先，你可能希望执行默认的深度链接处理
          // 检查应用是否通过深度链接打开
          const url = await Linking.getInitialURL();

          if (url != null) {
            return url;
          }

          // 处理来自 expo 推送通知的 URL
          const response = Notifications.getLastNotificationResponse();

          return response?.notification.request.content.data.url;
        },
        subscribe(listener) {
          const onReceiveURL = ({ url }: { url: string }) => listener(url);

          // 监听来自深度链接的传入链接
          const eventListenerSubscription = Linking.addEventListener('url', onReceiveURL);

          // 监听 expo 推送通知
          const subscription = Notifications.addNotificationResponseReceivedListener(response => {
            const url = response.notification.request.content.data.url;

            // 任意自定义逻辑，用于判断是否需要处理该 URL
            //...

            // 让 React Navigation 处理该 URL
            listener(url);
          });

          return () => {
            // 清理事件监听器
            eventListenerSubscription.remove();
            subscription.remove();
          };
        },
      }}>
      {/* 你的应用内容 */}
    </NavigationContainer>
  );
}
```

## 权限

### Android

- 在 Android 上，此模块需要获得订阅设备启动的权限。它用于在设备（重新）启动时设置计划通知。
  `RECEIVE_BOOT_COMPLETED` 权限会通过该库的 **AndroidManifest.xml** 自动添加。

- 从 Android 12（API level 31）开始，要安排在精确时间触发的通知，你需要将
  `<uses-permission android:name="android.permission.SCHEDULE_EXACT_ALARM"/>` 添加到 **AndroidManifest.xml**。
  了解更多关于[精确闹钟权限](https://developer.android.com/about/versions/12/behavior-changes-12#exact-alarm-permission)。

- 在 Android 13 上，应用用户必须通过由操作系统自动触发的权限提示来选择是否接收通知。
  在至少创建一个通知频道之前，不会显示此提示。必须先调用 `setNotificationChannelAsync`，
  再调用 `getDevicePushTokenAsync` 或 `getExpoPushTokenAsync` 才能获取推送令牌。你可以在[官方文档](https://developer.android.com/develop/ui/views/notifications/notification-permission#new-apps)中了解更多关于 Android 13 新通知权限行为的信息。

<AndroidPermissions permissions={['RECEIVE_BOOT_COMPLETED', 'SCHEDULE_EXACT_ALARM']} />

### iOS

不需要使用说明，请参见[与通知相关的权限](#fetch-information-about-notifications-related-permissions)。

### 解释 iOS 权限响应

在 iOS 上，发送通知的权限比 Android 上更细粒度一些。这就是为什么你应该依赖 `NotificationPermissionsStatus` 的 `ios.status` 字段，而不是根级的 `status` 字段。

该值将是以下之一，可通过 `Notifications.IosAuthorizationStatus` 访问：

- `NOT_DETERMINED`：用户尚未就应用是否允许安排通知做出选择
- `DENIED`：应用未被授权安排或接收通知
- `AUTHORIZED`：应用被授权安排或接收通知
- `PROVISIONAL`：应用被临时授权发布非打断式用户通知
- `EPHEMERAL`：应用被授权在有限时间内安排或接收通知

## 无头（后台）通知

请参阅[你需要了解的内容](/push-notifications/what-you-need-to-know)指南中关于无头后台通知的[定义](/push-notifications/what-you-need-to-know/#headless-background-notifications)。

要在应用处于后台或未运行时处理通知，你需要执行以下操作：

- 将 `expo-task-manager` 包添加到你的项目中。
- [配置后台通知](#background-notification-configuration)。
- 在你的应用代码中，设置一个[后台任务](#registertaskasynctaskname)，在收到通知时运行。

然后发送一个推送通知，其：

- 仅包含 `data` 键（不包含 `title`、`body`）
- 为 iOS 设置 `_contentAvailable: true` —— 参见 [Expo 推送通知服务负载格式](/push-notifications/sending-notifications/#message-request-format)

### 后台通知配置&ensp;<PlatformTags platforms={["ios"]} />

要在 iOS 上使用无头（后台）通知推送通知，`remote-notification` 值需要存在于应用 **Info.plist** 文件中的 `UIBackgroundModes` 数组里。

**如果你正在使用 [CNG](/workflow/continuous-native-generation/)**，请将配置插件的 [`enableBackgroundRemoteNotifications` 属性](#configurable-properties)设置为 true，预构建会自动应用正确的配置。

<Collapsible summary="在 iOS 上手动配置 UIBackgroundModes">

如果你没有使用 Continuous Native Generation（[CNG](/workflow/continuous-native-generation/)），或者你正在使用原生 iOS 项目，那么你需要在 **Expo.plist** 文件中添加以下内容：

```xml ios/project-name/Supporting/Expo.plist
<key>UIBackgroundModes</key>
<array>
  <string>remote-notification</string>
</array>
```

### 设置自定义通知声音

要为应用添加自定义推送通知声音，请将 `expo-notifications` 插件添加到你的 **app.json** 文件中，然后在 `sounds` 键下提供一个可用作自定义通知声音的本地音频文件路径数组。这些本地路径相对于你的项目而言是本地路径。

```json app.json
{
  "expo": {
    "plugins": [
      [
        "expo-notifications",
        {
          "sounds": ["local/path/to/mySoundFile.wav"]
        }
      ]
    ]
  }
}
```

构建应用后，这个文件数组即可用于 [`NotificationContentInput`](#notificationcontentinput) 和 [`NotificationChannelInput`](#notificationchannelinput)。
你只需要提供基础文件名。下面是使用上述配置的示例：

```ts
await Notifications.setNotificationChannelAsync('new_emails', {
  name: 'E-mail notifications',
  importance: Notifications.AndroidImportance.HIGH,
  sound: 'mySoundFile.wav', // 只提供基础文件名
});

await Notifications.scheduleNotificationAsync({
  content: {
    title: "You've got mail! 📬",
    sound: 'mySoundFile.wav', // 只提供基础文件名
  },
  trigger: {
    type: Notifications.SchedulableTriggerInputTypes.TIME_INTERVAL,
    seconds: 2,
    channelId: 'new_emails',
  },
});
```

### 管理交互式通知的通知类别

通知类别允许你创建交互式推送通知，这样用户就可以直接对传入的通知进行响应
，无论是通过按钮还是文本回复。类别定义了用户可执行的一组操作，然后这些操作通过在 [`NotificationContent`](#notificationcontent) 中指定 `categoryIdentifier` 应用于通知。

<ContentSpotlight
  src="/static/images/sdk/notifications/categories.png"
  className="max-w-[800px]"
  alt="Android 和 iOS 上通知类别的示意图"
/>

在 iOS 上，通知类别还允许你进一步自定义通知。对于每个类别，不仅可以设置用户可执行的交互操作，还可以配置诸如当用户为你的应用禁用通知预览时显示的占位文本等内容。

## 平台特定指南

### 处理通知渠道&ensp;<PlatformTags platforms={['android 8+']} />

从 Android 8.0（API 级别 26）开始，所有通知都必须分配到一个渠道。对于每个渠道，
你可以设置应用于该渠道中所有通知的视觉和听觉行为。
然后，用户可以更改这些设置，并决定来自你的应用的哪些通知渠道应当具有干扰性或完全可见，
正如 [Android 开发者文档](https://developer.android.com/training/notify-user/channels) 所述。

如果你没有指定通知渠道，`expo-notifications` 会为你创建一个回退渠道，名为 **Miscellaneous**。
我们建议你始终确保为应用设置带有说明性名称的合适渠道，并始终将通知发送到这些渠道。

> 对于不支持此功能的平台（Android 8.0（26）以下版本和 iOS），调用这些方法不会产生任何作用。

### 自定义通知图标和颜色&ensp;<PlatformTags platforms={['android']} />

你可以通过使用 [Expo Prebuild](/workflow/continuous-native-generation) 与 [`expo-notifications` 配置插件](#configurable-properties) 来为项目中的通知配置 `icon` 和 `color` 键。这些都是构建时设置，因此你需要使用 `eas build -p android` 或 `npx expo run:android` 重新编译原生 Android 应用才能看到更改。

对于通知图标，请确保你遵循 [Google 的设计指南](https://m2.material.io/design/iconography/product-icons.html#design-principles)
（图标必须是全白色且背景透明），否则它可能不会按预期显示。

你也可以直接在 [`NotificationContentInput`](#notificationcontentinput) 的 `color` 属性下，针对**每条通知**设置自定义通知颜色。
