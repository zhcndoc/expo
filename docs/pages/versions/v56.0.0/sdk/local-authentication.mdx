`expo-local-authentication` 允许你使用 Biometric Prompt（Android）或 FaceID 和 TouchID（iOS）通过指纹或面部扫描来验证用户身份。

## 已知限制

### iOS&ensp;<PlatformTags platforms={['ios']} />

Expo Go 不支持 iOS 的 FaceID 身份验证。你需要创建一个 [development build](/develop/development-builds/introduction/) 来测试 FaceID。

## 在 app 配置中进行配置

如果你在项目中使用 config plugins（[Continuous Native Generation (CNG)](/workflow/continuous-native-generation/)），你可以使用 `expo-local-authentication` 内置的 [config plugin](/config-plugins/introduction/) 进行配置。该插件允许你配置各种无法在运行时设置、且需要重新构建应用二进制文件才会生效的属性。如果你的应用**没有**使用 CNG，那么你需要手动配置该库。

<ConfigPluginExample>

```json app.json
{
  "expo": {
    "plugins": [
      [
        "expo-local-authentication",
        {
          "faceIDPermission": "Allow $(PRODUCT_NAME) to use Face ID."
        }
      ]
    ]
  }
}
```

</ConfigPluginExample>

<ConfigPluginProperties
  properties={[
    {
      name: 'faceIDPermission',
      platform: 'ios',
      description:
        '用于设置 [`NSFaceIDUsageDescription`](#permission-nsfaceidusagedescription) 权限提示信息的字符串。',
      default: '"Allow $(PRODUCT_NAME) to use Face ID"',
    },
  ]}
/>

<ConfigReactNative>

如果你没有使用 Continuous Native Generation ([CNG](/workflow/continuous-native-generation/))，或者你正在手动使用原生 **ios** 项目，那么你需要向 **ios/[app]/Info.plist** 添加 `NSFaceIDUsageDescription` 键：

```xml Info.plist
<key>NSFaceIDUsageDescription</key>
<string>Allow $(PRODUCT_NAME) to use FaceID</string>
```

</ConfigReactNative>

## 权限

### Android

以下权限会通过该库的 **AndroidManifest.xml** 自动添加：

<AndroidPermissions permissions={['USE_BIOMETRIC', 'USE_FINGERPRINT']} />

### iOS

以下使用说明键会被该库使用：

<IOSPermissions permissions={['NSFaceIDUsageDescription']} />
