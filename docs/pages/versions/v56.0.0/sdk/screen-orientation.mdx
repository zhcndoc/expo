屏幕方向是指图形在设备上绘制时所采用的方向。例如，下图中的设备在物理上处于竖直和水平两种方向，但其屏幕方向为纵向。有关物理设备方向，请参阅 [Device Motion](devicemotion.mdx) 的方向部分。

<ContentSpotlight
  alt="不同物理方向下的纵向模式。"
  src="/static/images/screen-orientation-portrait.jpg"
/>

在 Android 和 iOS 平台上，更改屏幕方向会覆盖任何系统设置或用户偏好。在 Android 上，更改屏幕方向时可以考虑用户偏好的方向。在 iOS 上，应用无法访问用户和系统设置，且对屏幕方向的任何更改都会覆盖现有设置。

> Web 仅有[有限支持](https://caniuse.com/#feat=deviceorientation)。

### 警告

Apple 在 iOS 9 中为 iPad 添加了对 _分屏视图_ 模式的支持。这改变了系统处理屏幕方向的方式。简单来说，在 iOS 上，除非你同时打开两个并排的应用，否则你的 iPad 始终处于横向模式。若要使用此模块锁定屏幕方向，你需要禁用对此功能的支持。有关 _分屏视图_ 模式的更多信息，请查看 [Apple 官方文档](https://support.apple.com/en-us/HT207582)。

## 在应用配置中进行配置

如果你在项目中使用了配置插件（[Continuous Native Generation (CNG)](/workflow/continuous-native-generation/)），可以使用 `expo-screen-orientation` 内置的 [config plugin](/config-plugins/introduction/) 来进行配置。该插件允许你配置各种无法在运行时设置的属性，并且需要构建新的应用二进制文件后才会生效。如果你的应用**没有**使用 CNG，那么你需要手动配置该库。

<ConfigPluginExample>

```json app.json
{
  "expo": {
    "ios": {
      "requireFullScreen": true
    },
    "plugins": [
      [
        "expo-screen-orientation",
        {
          "initialOrientation": "DEFAULT"
        }
      ]
    ]
  }
}
```

</ConfigPluginExample>

  properties={[

      default: 'undefined',

## 使用 Expo Router 为每个页面单独设置方向

如果你使用 [Expo Router](/router/introduction/)，可以通过 [`Stack.Screen`](/router/advanced/stack/) 上的 `orientation` 选项为每个页面单独设置方向。这由 `react-native-screens` 提供支持，也是在堆栈导航器中按页面设置方向的推荐方式。

```tsx src/app/_layout.tsx
import { Stack } from 'expo-router';

export default function Layout() {
  return (
    <Stack>
      <Stack.Screen name="index" options={{ orientation: 'portrait' }} />
      <Stack.Screen name="landscape" options={{ orientation: 'landscape' }} />
    </Stack>
  );
}
```
