### 配置插件设置（可选）

如果你正在使用 EAS Build，可以通过 `@stripe/stripe-react-native` [配置插件](/config-plugins/introduction/)完成大部分 Stripe 设置。要进行设置，只需像下面这样将配置插件添加到 **app.json** 或 **app.config.js** 的 `plugins` 数组中，然后重新构建应用。

```json app.json
{
  "expo": {
    /* @hide ... */ /* @end */
    "plugins": [
      [
        "@stripe/stripe-react-native",
        {
          "merchantIdentifier": string | string [],
          "enableGooglePay": boolean
        }
      ]
    ],
  }
}
```

- **merchantIdentifier**：仅限 iOS。这是[在此处获取的 Apple 商户 ID](https://docs.stripe.com/apple-pay?platform=react-native)。否则，Apple Pay 将无法按预期工作。如果你有多个 merchantIdentifiers，可以将它们放在一个数组中。
- **enableGooglePay**：仅限 Android。表示是否启用 Google Pay 的布尔值。默认为 `false`。

### 常见问题

#### 浏览器弹窗没有重定向回我的应用

如果你依赖重定向，就需要向 `initStripe` 传入 `urlScheme`。为确保始终使用正确的 `urlScheme`，请传入：

```js
import * as Linking from 'expo-linking';
import Constants from 'expo-constants';

urlScheme:
  Constants.appOwnership === 'expo'
    ? Linking.createURL('/--/')
    : Linking.createURL(''),
```

[`Linking.createURL()`](./linking/#createurloptions) 会确保你使用的是正确的 scheme，无论你是在 Expo Go 中运行还是在生产应用中运行。`'/--/'` 在 Expo Go 中是必要的，因为它表示其后的子字符串对应于深度链接路径，而不是应用自身路径的一部分。

#### iOS 上的 PaymentSheet 本地化

在 Android 上，`PaymentSheet` 的翻译会根据设备的语言设置自动检测。

在 iOS 上，你必须启用 `CFBundleAllowMixedLocalizations`，并在应用配置的 [`ios.infoPlist`](../config/app/#infoplist) 下使用 `CFBundleLocalizations` 添加首选语言：

```json app.json
{
  "expo": {
    "ios": {
      "infoPlist": {
        "CFBundleAllowMixedLocalizations": true,
        "CFBundleLocalizations": ["fr"]
        /* @hide ... */ /* @end */
      }
      /* @hide ... */ /* @end */
    }
  }
}
```

## 限制

### Google Pay

[Expo Go](https://expo.dev/go) **不**支持 Google Pay。要使用 Google Pay，你必须创建一个[开发构建](/develop/development-builds/create-a-build/)。这可以通过 [EAS Build](/build/introduction) 完成，或者在本地运行 `npx expo run:android`。

### Apple Pay

[Expo Go](https://expo.dev/go) **不**支持 Apple Pay。要使用 Apple Pay，你必须创建一个[开发构建](/develop/development-builds/create-a-build/)。这可以通过 [EAS Build](/build/introduction) 完成，或者在本地运行 `npx expo run:ios`。
