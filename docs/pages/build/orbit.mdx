[Expo Orbit](https://expo.dev/orbit) 适用于 macOS 和 Windows，可让你更快地安装并启动来自 EAS、本地文件的构建或更新，或者在模拟器和实体设备上运行 Snack 项目。

<ContentSpotlight file="orbit/basic-features.mp4" />

## 为什么选择 Orbit

在 Orbit 之前，从 EAS 安装构建或更新（在 Android 和 iOS 实体设备或模拟器/仿真器上）或在模拟器上运行 Snack 项目都需要手动操作。你必须运行 `eas build:run` 命令，并为所选设备选择一个构建，或者下载归档文件，然后将其拖放到模拟器中（在 iOS 的情况下）。此外，对于 Snack 项目，还需要额外步骤，包括在虚拟设备上安装 Expo Go、登录，然后从列表中选择 Snack。Orbit 让这些步骤尽可能无缝。

## 亮点

- 列出并启动模拟器，包括在没有音频的情况下运行 Android 仿真器。
- 一键在模拟器和真实设备上安装并启动来自 EAS 的构建。
- 在 Android 仿真器或 iOS 模拟器上[安装并打开来自 EAS 的更新](/review/with-orbit/)。
- 一键在你的模拟器中启动 Snack 项目。
- 使用 Finder 或将文件拖放到菜单栏应用中，从本地文件安装并启动应用。Orbit 支持任何 Android **.apk**、iOS Simulator 兼容的 **.app**，或 ad hoc 签名的应用。
- 查看来自你的 [EAS 仪表板](https://expo.dev) 的置顶项目，并快速启动你最新的构建。

## 安装

> **info** Orbit 依赖于 macOS 和 Windows 上的 Android SDK，以及仅在 macOS 上用于设备管理的 `xcrun`，这需要同时设置好 [Android Studio](/workflow/android-studio-emulator/) 和 [Xcode](/workflow/ios-simulator/)。

<Tabs>

<Tab label="macOS">

你可以通过 Homebrew 下载适用于 macOS 的 Orbit，或者直接从 [GitHub releases](https://github.com/expo/orbit/releases) 下载。

<Terminal cmd={['$ brew install expo-orbit']} />

如果你希望 Orbit 在登录时自动启动，请点击菜单栏中的 Orbit 图标，然后进入 **Settings**，并选择 **Launch on Login** 选项。

</Tab>

<Tab label="Windows">

> **important** 适用于 Windows 的 Orbit 处于 [beta](/more/release-statuses/#beta) 阶段，并且仅兼容 x64 和 x86 机器。未来将添加对其他架构的兼容性。

你可以直接从 [GitHub releases](https://github.com/expo/orbit/releases) 下载适用于 Windows 的 Orbit。

</Tab>

</Tabs>
