## 创建新的 Google 服务账号密钥

以下是在 EAS 中配置新的 Google 服务账号密钥以使用 FCM V1 发送 Android 通知的步骤。

<Step label="1">

在 [Firebase 控制台](https://console.firebase.google.com)中为你的应用创建一个新的 Firebase 项目。如果你已经为你的应用创建了 Firebase 项目，请继续下一步。

<ContentSpotlight
  alt="在 Firebase 控制台中创建项目"
  src="/static/images/creating-google-service-account/fcm-v1/new-service-account/01-new-firebase-project.png"
  className="max-w-[800px]"
/>

</Step>

<Step label="2">

在 Firebase 控制台中，打开你项目的 **项目设置** > [**服务账号**](https://console.firebase.google.com/project/_/settings/serviceaccounts/adminsdk)。

<ContentSpotlight
  alt="Firebase 控制台中的服务账号设置选项卡"
  src="/static/images/creating-google-service-account/fcm-v1/new-service-account/02-manage-service-accounts.png"
  className="max-w-[800px]"
/>

</Step>

<Step label="3">

点击 **生成新的私钥**，然后通过点击 **生成密钥** 确认。将包含私钥的 JSON 文件安全地保存起来。

<ContentSpotlight
  alt="在 Firebase 控制台中生成新的私钥"
  src="/static/images/creating-google-service-account/fcm-v1/new-service-account/03-generate-key.png"
  className="max-w-[800px]"
/>

</Step>

<Step label="4">

将 JSON 文件上传到 EAS，并为发送 Android 通知进行配置。可以使用 EAS CLI 或在 [EAS 仪表板](https://expo.dev) 中完成。

<Tabs tabs={["EAS CLI", "expo.dev"]}>

<Tab>

- 运行 `eas credentials`
- 选择 `Android` > `production` > `Google Service Account`
- 选择 `Manage your Google Service Account Key for Push Notifications (FCM V1)`
- 选择 `Set up a Google Service Account Key for Push Notifications (FCM V1)` > `Upload a new service account key`
- 如果你之前已将 JSON 文件存储在项目目录中，EAS CLI 会自动检测到该文件并提示你选择它。按 <kbd>Y</kbd> 继续。

> **注意**：将 JSON 文件添加到版本控制的忽略文件中（例如 **.gitignore**），以避免将其提交到仓库，因为它包含敏感数据。

</Tab>

<Tab>

- 在 **项目设置** 下，点击导航菜单中的 [**凭据**](https://expo.dev/accounts/[account]/projects/[project]/credentials)
- 对于 **Android**，点击 **添加应用标识符** 或选择一个已有的 **应用标识符**
- 在 **服务凭据** > **FCM V1 service account key** 下，点击 **添加服务账号密钥**

<ContentSpotlight
  alt="expo.dev 上的项目凭据页面"
  src="/static/images/creating-google-service-account/fcm-v1/new-service-account/04-upload-credential-1.png"
  className="max-w-[800px]"
/>

- 在 **上传新密钥** 下，上传你的 JSON 凭据并点击 **保存**

<ContentSpotlight
  alt="将服务账号密钥文件保存到项目中"
  src="/static/images/creating-google-service-account/fcm-v1/new-service-account/04-upload-credential-2.png"
  className="max-w-[480px]"
/>

</Tab>

</Tabs>

</Step>

<Step label="5">

在你的项目中配置 **google-services.json** 文件。从 Firebase 控制台下载它，并将其放在项目目录的根目录下。

此文件是你的 Android 应用注册到 FCM 所必需的。你可以将此文件提交到仓库，因为它包含来自 Firebase 项目的面向公众的标识符。

**注意**：如果 **google-services.json** 已经配置好，你可以跳过这一步。

<ContentSpotlight
  alt="从 Firebase Cloud Console 下载 google-services.json"
  src="/static/images/creating-google-service-account/fcm-v1/new-service-account/05-setup-google-services-json.png"
  className="max-w-[800px]"
/>

在 **app.json** 中，添加 [`expo.android.googleServicesFile`](/versions/latest/config/app/#googleservicesfile)，其值为 **google-services.json** 的路径。

```json app.json
{
  "expo": {
  /* @hide ...*/ /* @end */
  "android": {
    /* @hide ...*/ /* @end */
    "googleServicesFile": "./path/to/google-services.json"
  }
}
```

</Step>

<Step label="6">

一切就绪！现在你可以使用 FCM V1 协议，通过 Expo Push Notifications 向 Android 设备发送通知了。

<ContentSpotlight
  alt="服务账号密钥已成功上传"
  src="/static/images/creating-google-service-account/fcm-v1/new-service-account/06-upload-credential-complete.png"
  className="max-w-[800px]"
/>

</Step>

## 使用现有的 Google 服务账号密钥

<Step label="1">

在 Google Cloud 控制台中打开 [IAM 管理页面](https://console.cloud.google.com/iam-admin/iam?authuser=0)。在 Permissions 选项卡中，找到你要修改的 **Principal**，然后点击 **Edit Principal** 的铅笔图标。

<ContentSpotlight
  alt="Google Cloud 控制台 IAM 管理页面中的 Edit Principal"
  src="/static/images/creating-google-service-account/fcm-v1/existing-service-account/01-iam-admin-page.png"
  className="max-w-[800px]"
/>

</Step>

<Step label="2">
点击 **Add Role**，并从下拉菜单中选择 **Firebase Messaging API Admin** 角色。点击 **Save**。

<ContentSpotlight
  alt="分配角色"
  src="/static/images/creating-google-service-account/fcm-v1/existing-service-account/02-add-role-1.png"
  className="max-w-[800px]"
/>

<ContentSpotlight
  alt="选择 Firebase Messaging API Admin 角色"
  src="/static/images/creating-google-service-account/fcm-v1/existing-service-account/02-add-role-2.png"
  className="max-w-[800px]"
/>

<ContentSpotlight
  alt="保存角色分配"
  src="/static/images/creating-google-service-account/fcm-v1/existing-service-account/02-add-role-3.png"
  className="max-w-[800px]"
/>

</Step>

<Step label="3">

你需要指定给 EAS 用于发送 FCM V1 通知的 JSON 凭据文件，可通过 EAS CLI 或在 [EAS 仪表板](https://expo.dev) 中完成。你可以上传新的 JSON 文件或选择之前上传过的文件。

<Tabs tabs={["EAS CLI", "expo.dev"]}>

<Tab>

- 运行 `eas credentials`
- 选择 `Android` > `production` > `Google Service Account`
- 选择 `Manage your Google Service Account Key for Push Notifications (FCM V1)`
- 选择 `Set up a Google Service Account Key for Push Notifications (FCM V1)` > `Upload a new service account key`
- EAS CLI 会自动检测本地机器上的文件，并提示你选择它。按 <kbd>Y</kbd> 继续。

> **注意**：将 JSON 文件添加到版本控制的忽略文件中（例如 **.gitignore**），以避免将其提交到仓库，因为它包含敏感数据。

</Tab>

<Tab>

- 在 **项目设置** 下，点击导航菜单中的 [**凭据**](https://expo.dev/accounts/[account]/projects/[project]/credentials)
- 对于 **Android**，点击 **添加应用标识符** 或选择一个已有的 **应用标识符**
- 在 **服务凭据** > **FCM V1 service account key** 下，点击 **添加服务账号密钥**

<ContentSpotlight
  alt="expo.dev 上的项目凭据页面"
  src="/static/images/creating-google-service-account/fcm-v1/new-service-account/04-upload-credential-1.png"
  className="max-w-[800px]"
/>

- 在 **上传新密钥** 下，上传你的 JSON 凭据并点击 **保存**

<ContentSpotlight
  alt="将服务账号密钥文件保存到项目中"
  src="/static/images/creating-google-service-account/fcm-v1/new-service-account/04-upload-credential-2.png"
  className="max-w-[480px]"
/>

</Tab>

</Tabs>

</Step>

<Step label="4">

在你的项目中配置 **google-services.json** 文件。从 Firebase 控制台下载它，并将其放在项目目录的根目录下。

此文件是你的 Android 应用注册到 FCM 所必需的。你可以将此文件提交到仓库，因为它包含来自 Firebase 项目的面向公众的标识符。

在 **app.json** 中，添加 [`expo.android.googleServicesFile`](/versions/latest/config/app/#googleservicesfile)，其值为 **google-services.json** 的路径。

  "expo": {
