本页面涵盖有关 [Expo Application Services (EAS)](/eas/) 的计划、账单和付款的常见问题。

## 计划

### 我该如何更新我的计划？

要更新你的 Organization 账户的计划，请确保你拥有 [Owner 或 Admin 角色权限](/accounts/account-types/#manage-access)。对于 Personal 账户，你始终拥有 **Owner** 角色。更多信息请参见 [更改成员的角色](/accounts/account-types/#change-the-role-of-a-member)。

确认你的角色后，请参见 [升级到新计划](/billing/manage/#upgrade-to-a-new-plan) 进行升级，或参见 [降级计划](/billing/manage/#downgrade-a-plan) 以降级现有计划。

### 我该如何取消计划？

更多信息请参见 [取消计划](/billing/manage/#cancel-a-plan)。

### 如果我从错误的账户订阅了怎么办？

如果你从错误的账户订阅了某个计划：

- 在 EAS 仪表板侧边栏中，点击顶部的账户切换器并选择你打算订阅的账户。
- 前往 [Billing](https://expo.dev/settings/billing)，在 **Current Plan** 下， [按照步骤订阅正确的计划](/billing/manage/#upgrade-to-a-new-plan)。
- 在错误的账户下，前往 **[Receipts](https://expo.dev/accounts/[account]/settings/receipts)** 并发起 [退款申请](/billing/invoices-and-receipts/#request-a-refund)。

### 我使用的是 Free 计划，只需要少量额外的构建或更新

如果你使用的是 Free 计划，并且已经完成了每月免费构建和更新配额，请升级到 [Starter 计划](/billing/plans/)。每月 19 美元，你可以获得 45 美元的构建额度，以及 EAS Update 的 3,000 个每月活跃用户（相比之下 Free 计划为 1,000）。当你的需求得到满足后，你可以 [从 Starter 计划降级回 Free 计划](/billing/manage/#cancel-a-plan)。

要从 Free 计划升级到 Starter 计划，请参见 [升级到新计划](/billing/manage/#upgrade-to-a-new-plan)。

要从 Starter 计划降级到 Free 计划，请参见 [取消计划](/billing/manage/#cancel-a-plan)。

### 我的付费计划中的 EAS Build 额度已经用完了。我可以降级到 Free 计划来使用免费构建额度吗？

不可以。如果你订阅的是付费计划并且用完了包含的 EAS Build 额度，额外的构建将按照 [按使用量计费](/billing/usage-based-pricing/) 收费。

如果你取消订阅，Free 计划会在当前计费周期结束后生效。一旦付费订阅结束且你的账户处于 Free 计划下，你就可以使用 Free 计划的每月配额（受其限制和重置时间表约束）。更多详情请参见 [取消计划](/billing/manage/#cancel-a-plan)。

### 升级到付费订阅时，我可以转移未使用的免费计划额度吗？

不可以，你的免费计划额度不能转移到其他订阅计划。所有付费计划都提供额度，以便为 EAS Build 提供优先构建，并通过更多每月活跃用户和额外带宽扩展对 EAS Update 的访问。

## 账单

### 某个计划的计费周期什么时候开始？

对于 Free 计划，计费周期从每个月的第一天开始。

对于 [所有付费计划](/billing/plans/#plans)，计费周期从该计划的订阅日期开始。

### 我该如何更新账单信息或添加税务 ID？

要更新你的 Organization 账户的账单信息或添加税务 ID，请确保你拥有 [Owner 或 Admin 角色](/accounts/account-types/#manage-access)。对于 Personal 账户，你始终拥有 **Owner** 角色。确认你的角色后：

- 前往你的 [账户 Billing](https://expo.dev/settings/billing)，然后在右侧边栏中点击 **Manage billing**。这将带你进入 Stripe 的门户。
- 在 Stripe 的门户中，在 **Billing information** 下，点击 **Update information**，即可更新与账单相关的信息，例如账单名称、邮箱、地址和税务 ID。

更多详情请参见 [管理账单信息](/billing/manage/#manage-billing-information)。

### 我可以更新上一张发票上的账单信息吗？

不可以。更新账单信息只会反映在下一张发票上。

### 你们可以把收据通过电子邮件发给我吗？

不可以。你账户的 Owner 或 Admin 可以下载它们。更多信息请参见 [下载发票](/billing/invoices-and-receipts/#download-and-view-an-invoice)。

### 我该如何通过使用 EAS Update 来减少 EAS Build 的使用量？

使用 [EAS Update](/eas-update/introduction/) 和 [development builds](/develop/development-builds/introduction/) 来测试和部署新代码，而无需创建新构建。对于大多数应用来说，这个选项更好，因为 JavaScript 代码的变更比底层原生代码更频繁。你可以使用 EAS Update 创建多个测试渠道，从而减少团队创建额外构建的需要。

你也可以通过 [EAS Workflows 中的 fingerprint](/eas/workflows/examples/deploy-to-production/) 仅在 Android 和 iOS 项目的原生代码发生变化时创建构建。否则，对于仅 JavaScript 的变更，工作流将跳过创建新构建，并通过 EAS Update 发布更新。

更多信息请参见 [如何优化构建使用量](/billing/usage-based-pricing/#how-to-optimize-build-usage)。

### 我如何知道自己何时接近使用限制？

当你的账户达到计划中包含的 EAS Build 额度的 80% 和 100% 时，Expo 会自动向账户的 [Owners 和 Admins](/accounts/account-types/#manage-access) 发送电子邮件通知。更多详情请参见 [使用通知](/billing/usage-based-pricing/#usage-notifications)。

### 我该如何估算下一张账单？

要估算你的下一张账单，请前往 [Billing](https://expo.dev/settings/billing) 并查看 **Usage** 部分。你会看到基于 [resource class](/build/eas-json/#selecting-resource-class) 的 EAS Build 使用情况、基于每月活跃用户和全球边缘带宽的 EAS Update 使用情况，以及两者的花费金额。

更多信息请参见 [按使用量计费如何运作](/billing/usage-based-pricing/#how-usage-based-pricing-works)。

### 什么是 EAS Update 每月活跃用户（MAU）？

每月活跃用户（MAU）是指在单个按月计费周期内，通过 EAS Update 下载至少一个更新的你应用中的唯一用户。更多信息请参见 [某个计费周期内每月活跃用户如何统计](/eas-update/introduction/#how-are-monthly-active-users-counted-for)。

## 付款

### 我可以按年付款吗？

[Enterprise 计划](/billing/plans/#enterprise) 客户可以使用年度计划。更多信息请 [联系我们](https://expo.dev/contact)。

### 我该如何更新我们的付款信息？

要更新你的 Organization 账户的付款信息，请确保你拥有 [Owner 或 Admin 角色](/accounts/account-types/#manage-access)。对于 Personal 账户，你始终拥有 **Owner** 角色。确认你的角色后：

- 前往你的 [账户 **Billing**](https://expo.dev/settings/billing)，然后在右侧边栏中点击 **Manage billing**。这将带你进入 Stripe 的门户。
- 在 Stripe 的门户中，在 **Payment method** 下，点击 **Add payment method** 添加新的付款方式。

更多详情请参见 [管理账单信息](/billing/manage/#payment-method)。

### 我可以通过自动清算所（ACH）或银行/电汇付款吗？

[Enterprise 计划](/billing/plans/#enterprise) 的年度计划客户可以使用 ACH 作为信用卡付款的替代方式。更多信息请 [联系我们](https://expo.dev/contact)。

### 我需要 W-9 或其他法律文件

如需申请 W-9，请 [联系我们](https://expo.dev/contact)。

请在 [expo.dev/terms](https://expo.dev/terms) 查看我们的法律条款。

### Expo 会存储我的卡信息吗？

不会，Expo 不会。我们使用 Stripe 来处理支付系统，由 Stripe 存储。更多信息请参见 [Stripe 如何处理安全性](https://docs.stripe.com/security)。

### 我这个月为一次大型构建支付了多少费用？

要查看一次大型构建的费用，请前往 [Billing](https://expo.dev/settings/billing) 并查看 **Usage** 部分。你会看到基于 [resource class](/build/eas-json/#selecting-resource-class) 的 EAS Build 使用情况以及花费金额的摘要。

## 附加项

### 我该如何增加我账户中的构建并发数？

如果你已经订阅了付费的 EAS 计划，可以在 **Billing** 下的 [**Add-ons**](https://expo.dev/settings/billing) 部分购买额外的并发数。

如果你使用的是 Free 计划，则需要先设置付费订阅。 [点击这里](https://expo.dev/accounts/[account]/settings/billing/cart) 选择一个新计划。然后，在结账页面选择要添加到订阅中的额外并发数。

每个计划包含的并发数不同。如果你需要超过 5 个额外并发数，请 [联系我们](https://expo.dev/contact)。
