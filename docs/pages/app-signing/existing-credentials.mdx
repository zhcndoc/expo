EAS Build 为你提供两种方式来向构建任务提供应用签名凭据：

1. [自动管理的凭据](/app-signing/managed-credentials/): EAS 可以托管你的应用签名凭据，并负责与拥有必要权限的队友共享它们。
2. [本地凭据](/app-signing/local-credentials/): 你在项目中创建一个 **credentials.json** 文件，指向你的 keystore（Android）和/或 provisioning profile 以及 distribution certificate（iOS），并附带相关密码。每次运行某个构建任务时，它都会从你的本地机器上传，在该构建任务完成后即被清除。

无论你选择哪种选项，使用你现有凭据集的第一步都是先将它们设置为 **credentials.json** 中的本地凭据。有关如何执行此操作的更多信息，请参阅[本地凭据指南中的 credentials.json 部分](/app-signing/local-credentials/#credentialsjson)。

一旦你的 **credentials.json** 文件配置完成，你就可以运行 `eas credentials`，选择一个平台，然后选择 `"使用 credentials.json 中的值更新 Expo 服务器上的凭据"` 将它们上传，由 EAS 托管和管理（如果你愿意的话）。[阅读更多关于同步凭据的内容](/app-signing/syncing-credentials/)。
