本指南概述了如何从你的电脑或 CI/CD 服务将应用提交到 Apple App Store。

## 从你的电脑提交应用

<Prerequisites numberOfRequirements={4}>
  <Requirement number={1} title="注册 Apple Developer 账号">
    提交应用到 Apple App Store 需要一个 Apple Developer 账号。你可以在 [Apple Developer Portal](https://developer.apple.com/account/) 上注册 Apple Developer 账号。
  </Requirement>
  <Requirement number={2} title="在 app.json 中包含 bundle 标识符">
    在 **app.json** 中包含你应用的 bundle 标识符：

    ```json app.json
    {
      "ios": {
        "bundleIdentifier": "com.yourcompany.yourapp"
      }
    }
    ```

  </Requirement>
  <Requirement number={3} title="安装 EAS CLI 并使用你的 Expo 账号进行身份验证">
    安装 EAS CLI 并使用你的 Expo 账号登录：
    
    <Terminal cmd={['$ npm install -g eas-cli && eas login']} />
  </Requirement>
  <Requirement number={4} title="构建生产版应用">
    你需要一个可用于商店提交的生产构建。你可以使用 [EAS Build](/build/introduction/) 来创建一个：
    
    <Terminal cmd={['$ eas build --platform ios --profile production']} />

    另外，你也可以使用 `eas build --platform ios --profile production --local` 在你自己的电脑上构建应用，或者使用 Xcode。

  </Requirement>
</Prerequisites>

完成所有前置条件后，你就可以开始提交流程了。

运行以下命令将构建提交到 Apple App Store：

<Terminal cmd={['$ eas submit --platform ios']} />

该命令会逐步引导你完成应用提交过程。你可以通过在 **eas.json** 中添加提交配置文件来配置提交流程：

```json eas.json
{
  "submit": {
    "production": {
      "ios": {
        "ascAppId": "your-app-store-connect-app-id"
      }
    }
  }
}
```

<Collapsible summary={<>如何查找 <CODE>ascAppId</CODE></>}>
  1. 登录 [App Store Connect](https://appstoreconnect.apple.com/) 并选择你的团队。
  2. 前往 [Apps](https://appstoreconnect.apple.com/apps)。
  3. 点击你的应用。
  3. 确保 **App Store** 选项卡处于激活状态。
  4. 在左侧面板的 **General** 部分下，选择 **App Information**。
  5. 你的应用的 `ascAppId` 可以在 **General Information** 部分下的 **Apple ID** 中找到。

  <ContentSpotlight
    alt="App Store Connect 中的 Apple ID 和 Apple Team ID"
    src="/static/images/eas-submit/finding-ascAppId.png"
    className="max-w-[800px]"
  />
</Collapsible>

了解你可以在 [eas.json 参考](/eas/json/#ios-specific-options-1) 中提供的所有选项。

为了加快提交流程，你可以使用 `--auto-submit` 标志，在构建完成后自动提交：

<Terminal cmd={['$ eas build --platform ios --auto-submit']} />

在 [自动化提交](/build/automate-submissions/) 指南中了解有关 `--auto-submit` 标志的更多信息。

完成所有前置条件后，你就可以设置 CI/CD 流水线，将你的应用提交到 Apple App Store。

### 使用 EAS Workflows CI/CD

你可以使用 [EAS Workflows](/eas/workflows/get-started/) 自动构建并提交你的应用。

1. 在项目根目录下创建一个名为 **.eas/workflows/submit-ios.yml** 的工作流文件。
2. 在 **submit-ios.yml** 中，你可以使用以下工作流来启动一个提交 iOS 应用的任务：

   ```yaml .eas/workflows/submit-ios.yml
   on:
     push:
       branches: ['main']

   jobs:
     build_ios:
       name: Build iOS app
       type: build
       params:
         platform: ios
         profile: production

     # @info #
     submit_ios:
       name: Submit to TestFlight
       needs: [build_ios]
       type: testflight
       params:
         build_id: ${{ needs.build_ios.outputs.build_id }}
     # @end #
   ```

   上面的工作流会构建 iOS 应用，然后将其提交到 Apple App Store 的 TestFlight。你可以使用 `testflight` 任务与内部和外部测试组共享。更多信息请参见 [预置的 `testflight` 任务](/eas/workflows/pre-packaged-jobs/#testflight)。

### 使用其他 CI/CD 服务

你可以使用其他 CI/CD 服务通过 EAS Submit 提交应用，例如 GitHub Actions、GitLab CI 等，只需运行以下命令：

<Terminal cmd={['$ eas submit --platform ios --profile production']} />

该命令需要一个 [个人访问令牌](/accounts/programmatic-access/#personal-access-tokens) 来使用你的 Expo 账号进行身份验证。一旦你拥有了它，请在 CI/CD 服务中提供 `EXPO_TOKEN` 环境变量，这样 `eas submit` 命令就可以运行。

## 手动提交

如果你需要在不使用 EAS Submit 的情况下提交构建，例如该服务因维护暂时不可用时，你可以从 macOS 设备手动上传到 Apple App Store。

<Collapsible summary="如何从 macOS 设备手动上传到 Apple App Store">

#### 在 App Store Connect 中创建条目

如果你还没有，请先在 App Store Connect 中创建一个应用资料：

1. 前往 [App Store Connect](https://appstoreconnect.apple.com) 并登录。确保你已接受页面顶部的任何法律声明或条款。
2. 点击 Apps 标题旁边的蓝色加号按钮，然后点击 **New App**。
3. 添加你应用的名称、语言、bundle 标识符和 SKU（终端用户看不到这个，它可以是任何唯一字符串。一个常见选择是你应用的 bundle 标识符，例如 "com.company.my-app"）。
4. 点击 **Create**。如果成功，那么你就创建了你的应用记录。

#### 使用 Transporter 上传

最后，你需要将 IPA 上传到 Apple App Store。

1. 从 [App Store 下载 **Transporter**](https://apps.apple.com/app/transporter/id1450874784)。
2. 使用你的 Apple ID 登录。
3. 通过将 IPA 文件直接拖入 Transporter 窗口，或者通过 **+** 或 **Add App** 按钮打开的文件对话框中选择它来添加构建。
4. 点击 **Deliver** 按钮提交。

这个过程可能需要几分钟，然后还需要在苹果服务器上处理 10-15 分钟。之后，你可以在 App Store Connect 中查看你的二进制文件状态：

1. 访问 [App Store Connect](https://appstoreconnect.apple.com)，选择 **My Apps**，然后点击你之前创建的应用条目。
2. 向下滚动到 **Build** 部分，并选择你刚刚上传的二进制文件。

</Collapsible>
