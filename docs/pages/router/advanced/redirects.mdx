> **warning** 静态重定向可从 Expo Router `4.x.x` 开始使用。

## Redirect 组件

要使用 `<Redirect />` 组件，请从 `expo-router` 中导入它，并使用 `href` 属性指定所需的目标路由。

```tsx
import { Redirect } from 'expo-router';

export default function Index() {
  return <Redirect href="/home" />;
}
```

在上面的示例中，当应用用户访问此页面时，他们会被自动重定向到 `/home` 路由。

你可以在组件中使用 `<Redirect />`，在导航前先检查条件。

```tsx
import { Redirect } from 'expo-router';
import { useState, useEffect } from 'react';

export default function ProtectedPage() {
  const [isAuthenticated, setIsAuthenticated] = useState(false);

  useEffect(() => {
    // 模拟检查身份验证状态
    setTimeout(() => setIsAuthenticated(true), 2000);
  }, []);

  if (!isAuthenticated) {
    return <Redirect href="/login" />;
  }

  return <Text>欢迎来到受保护页面！</Text>;
}
```

在这种情况下，如果用户未通过身份验证，他们将被重定向到 `/login` 页面。

## 静态重定向

静态重定向允许你使用 `expo-router` 配置插件在 [应用配置](/workflow/configuration/) 中指定重定向配置。这使你能够为 **app** 目录中可能不再存在的路由指定重定向。当你导航到某个屏幕时，Expo Router 会模拟向本地服务器发起 `GET` 请求，服务器会返回该屏幕的内容。这确保了 Expo Router 在原生端、Web 端以及获取 React Server Components 时都能保持一致的行为。因此，这种配置基于 **服务器请求**，并且基于请求 URL，而不是屏幕名称。

```json app.json
{
  "plugins": [
    [
      "expo-router",
      {
        "redirects": [
          {
            "source": "/redirect/from/here",
            "destination": "/to/this/route"
          },
          {
            "source": "/or/redirect/from/here",
            "destination": "http://to.this.site"
          }
        ]
      }
    ]
  ]
}
```

每个重定向配置都应包含以下属性：

- **source**：传入请求的路径模式
- **destination**：你希望路由到的路径。这可以是 Expo Router 项目中的路径，也可以是外部 URL。

重定向配置可以包含以下可选属性：

- **permanent**：接受 `true` 或 `false`。如果为 `true`，将使用 308 状态码，指示客户端/搜索引擎永久缓存该重定向。如果为 `false`，将使用 307 状态码，它是临时的且不会被缓存。

与 **app** 目录中的路由不同，目录路由不需要添加 `/index` 后缀，API 路由也不需要添加 `+api`。你不能为 **_layout** 文件创建重定向。

### 动态路由

`source` 和 `destination` 路由可以使用 [动态路由语法](/develop/app-navigation/) 来为动态路由创建重定向。你可以使用 `expo-router` 配置插件在 **app.json** 中定义它们。

```json app.json
{
  "plugins": [
    [
      "expo-router",
      {
        "redirects": [
          {
            "source": "/redirect/[slug]",
            "destination": "/target/[slug]"
          }
        ]
      }
    ]
  ]
}
```

在执行重定向时，如果可用，变量名将会匹配。任何未匹配的变量都将作为查询参数传递。

```json app.json
{
  "plugins": [
    [
      "expo-router",
      {
        "redirects": [
          {
            "source": "/redirect/[fruit]/[vegetable]/[meat]",
            "destination": "/target/[vegetable]/[fruit]"
          }
        ]
      }
    ]
  ]
}
```

使用上面的配置，`/redirect/apple/carrot/beef` 将重定向到 `/target/carrot/apple?meat=beef`

### HTTP 方法

若只想重定向某些 HTTP 方法，请在数组中以字符串形式提供这些方法。请记住，屏幕会被视为一次 `GET` 请求。

```json app.json
{
  "plugins": [
    [
      "expo-router",
      {
        "redirects": [{
          "source": "/redirect/[slug]",
          "destination": "/target/[slug]"
          "methods": ["POST"]
        }]
      }
    ]
  ]
}
```

## 重写

> **warning** 静态 URL 重写仅支持服务器环境，
> 在静态导出或原生应用中**不可用**。对于客户端
> 导航，请考虑改用 [重定向](#redirect-component)。

重写只能通过静态配置实现。它与重定向不同，因为它作为 URL 代理并隐藏目标 URL。重定向会返回 HTTP 状态码并重新路由请求，而重写会在不更改 URL 的情况下渲染新的目标。

```json app.json
{
  "plugins": [
    [
      "expo-router",
      {
        "rewrites": [
          {
            "source": "/redirect/from/here",
            "destination": "/to/this/route"
          }
        ]
      }
    ]
  ]
}
```
