EAS Hosting 是一项 React 托管服务，可让你将导出的 Expo Web 构建部署到预览或生产 URL。

本指南将带你完成创建第一个 Web 部署的过程。

## 前置条件

<Collapsible summary="一个 Expo 用户账户">

无论你是否订阅了 EAS 付费方案，还是使用免费计划，任何拥有 Expo 账户的人都可以使用 EAS Hosting。你可以在 [expo.dev/signup](https://expo.dev/signup) 注册。

付费订阅者可以创建更多部署，拥有更多带宽、存储和请求，并且可以设置自定义域名。你可以在 [EAS pricing](https://expo.dev/pricing#host) 了解不同方案和权益。

</Collapsible>

<Collapsible summary="一个 Expo Router Web 项目">

还没有项目？没问题。你可以快速轻松地创建一个可用于本指南的“Hello world”应用。

运行以下命令创建一个新项目：

<Terminal cmd={['$ npx create-expo-app@latest my-app --template default@sdk-55']} />

</Collapsible>

<Step label="1">
## 安装最新的 EAS CLI

EAS CLI 是你在终端中与 EAS 服务交互时使用的命令行应用。要安装它，请运行以下命令：

<Terminal cmd={['$ npm install --global eas-cli']} />

你也可以使用上面的命令来检查是否有可用的新版本 EAS CLI。我们建议你始终保持为最新版本。

> 我们建议在全局包安装时使用 `npm` 而不是 `yarn`。你也可以改用 `npx eas-cli@latest`。在文档中凡是需要使用 `eas` 的地方，请记得使用那个命令替代。

</Step>

<Step label="2">
## 登录你的 Expo 账户

如果你已经通过 Expo CLI 登录了 Expo 账户，可以跳过本节所述步骤。如果还没有，请运行以下命令进行登录：

<Terminal cmd={['$ eas login']} />

你可以通过运行 `eas whoami` 来检查自己是否已登录。

</Step>

<Step label="3">
## 准备你的项目

对于应用配置文件中的 [`expo.web.output`](/versions/latest/config/app/#output)，请决定将其设置为 `single`、`static` 还是 `server`。

- `single`：将你的 Expo 应用导出为单页应用，仅输出一个 `index.html`
- `static`：将你的 Expo 应用导出为一个[静态生成的 Web 应用](/router/web/static-rendering/)
- `server`：支持[服务器函数](/guides/server-components/#react-server-functions)和[API 路由](/router/web/api-routes/)，同时也支持应用的静态页面

> 如果你不确定需要哪种输出模式，不用担心，你之后随时可以更改这个值并重新部署。

</Step>

<Step label="4">
### 导出你的应用

你需要将你的 Web 项目导出到一个 **dist** 目录。为此，请运行：

<Terminal cmd={['$ npx expo export --platform web']} />

> 记得在每次部署前都重新运行此命令。

</Step>

<Step label="5">

### 部署你的应用

</Step>

现在将你的网站发布到 EAS Hosting：

<Terminal cmd={['$ eas deploy']} />

你第一次运行此命令时，它会：

1. 提示你连接一个 EAS 项目（如果你还没有这样做）
2. 要求你选择一个预览子域名

> **info** **预览子域名** 是用于你应用预览 URL 的前缀。
> 例如，如果你选择 `my-app` 作为预览子域名，那么你的预览 URL 可能会是这样：`https://my-app--or1170q9ix.expo.app/`，而你的生产 URL 将会是：`https://my-app.expo.app/`。

部署完成后，CLI 将输出你已部署应用可访问的预览 URL，以及指向 EAS Dashboard 中部署详情的链接。
