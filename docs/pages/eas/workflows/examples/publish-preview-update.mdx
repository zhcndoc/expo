一旦你对项目进行了更改，你就可以通过发布一个 [预览更新](/review/share-previews-with-your-team/) 来与团队共享更改的预览。当你想与团队一起审查更改，而无需拉取最新更改并在本地运行时，这会很有用。

你可以在开发构建 UI 中以及通过 EAS 仪表板上的可扫描 QR 码访问预览更新。当在每次提交时发布预览时，你的团队可以在无需拉取最新更改并在本地运行的情况下审查更改。

## 开始使用

<Prerequisites>
  <Requirement title="设置 EAS Update">
    你的项目需要先完成 [EAS Update](/eas-update/introduction/) 设置，才能发布预览更新。你可以通过以下方式为项目进行设置：

    <Terminal cmd={['$ eas update:configure']} />

  </Requirement>
  <Requirement title="创建新的开发构建">
    在为项目完成配置后，请为每个平台创建新的 [开发构建](/develop/development-builds/create-a-build/)。
  </Requirement>
</Prerequisites>

以下工作流会为每个分支上的每次提交发布一个预览更新。

```yaml .eas/workflows/publish-preview-update.yml
name: 发布预览更新

on:
  push:
    branches: ['*']

jobs:
  publish_preview_update:
    name: 发布预览更新
    type: update
    params:
      branch: ${{ github.ref_name || 'test' }}
```
