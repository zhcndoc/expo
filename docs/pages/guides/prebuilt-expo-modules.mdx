---
title: 预构建 Expo 模块
description: 了解预构建的 Expo 模块如何缩短 Android 和 iOS 的原生构建时间。
---

import { Collapsible } from '~/ui/components/Collapsible';
import { Terminal } from '~/ui/components/Snippet';

本地构建时间可能会拖慢你的开发流程。Expo 为其最复杂的一些模块提供了预构建版本，因此你的项目链接的是预编译二进制文件，而不是在每次构建时都从源代码重新编译它们。在 Android 上，这些二进制文件以通过 Gradle 连接的 `.aar` 文件形式提供；在 iOS 上，它们以通过 CocoaPods 连接的 `XCFrameworks` 形式提供。两者都打包在常规的 Expo npm 包中，而尚未预编译的包会自动回退到从源代码构建——预编译模块和源码构建模块可以在同一个项目中共存。

**大多数项目不需要做任何事情** —— 在新项目和现有项目中，只要 SDK 版本受支持，预构建的 Expo Modules 就会自动启用。

- **Android**：自 SDK 53 起默认启用。
- **iOS**：在 SDK 56+ 中默认启用。在 SDK 55 中，仅在 EAS Build 上默认启用——在你的 shell 中设置 `EXPO_USE_PRECOMPILED_MODULES=1` 即可在本地构建时启用。

<Collapsible summary="在 iOS 上禁用">

将 `EXPO_USE_PRECOMPILED_MODULES` 设置为 `0`。对于本地构建，请在你的 shell 中导出该环境变量。

对于 EAS Build，创建一个 [EAS 环境变量](/eas/environment-variables/manage/)：

<Terminal
  cmd={['$ eas env:create --name EXPO_USE_PRECOMPILED_MODULES --value 0 --visibility plaintext']}
/>

CLI 会提示你选择该变量适用的环境（`development`、`preview`、`production`）。

</Collapsible>

<Collapsible summary="通过 Expo Autolinking 禁用特定模块">

在 **package.json** 中使用 `buildFromSource` 配置 Expo Autolinking。使用 `".*"` 可对所有预构建模块停用，或者列出特定的包名。此设置同时适用于 `android` 和 `ios`：

{/* prettier-ignore */}
```json package.json
{
  "name": "your-app-name",
  "expo": {
    "autolinking": {
      "android": {
        "buildFromSource": [".*"]
      },
      "ios": {
        "buildFromSource": [".*"]
      }
    }
  }
}
```

这通常只会在你自己修改模块源代码时才需要。

</Collapsible>
