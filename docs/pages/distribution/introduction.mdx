通过将应用提交到应用商店，或使用 [内部分发](/build/internal-distribution)，让用户使用你的应用。

<Terminal
  cmd={[
    '# 安装 CLI',
    '$ npm i -g eas-cli',
    '',
    '# 构建并提交你的应用',
    '$ eas build --auto-submit',
    '',
    '# 或者 -- 提交现有二进制文件',
    '$ eas submit',
  ]}
  cmdCopy="npm i -g eas-cli && eas build --auto-submit"
/>

你可以使用 [EAS CLI](/eas) 运行 `eas build --auto-submit` 来构建你的应用，并自动上传二进制文件以分发到 Google Play 商店和 Apple App Store。

这会为任何 React Native 应用自动管理 Android 和 iOS 的**所有原生代码签名**。支付、通知、通用链接和 iCloud 等高级功能可以根据你的 [配置插件](/config-plugins/introduction/) 或原生权限自动启用，这意味着不再需要在缓慢的控制台中费力配置库以确保它们正确设置。
