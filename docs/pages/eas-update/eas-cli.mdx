EAS Update 的工作方式是将 _分支_ 关联到 _渠道_。渠道在构建时指定，并存在于构建的原生代码中。分支是一个按顺序排列的更新列表，类似于 Git 分支，而 Git 分支是一个按顺序排列的提交列表。借助 EAS Update，我们可以将任意渠道链接到任意分支，从而让不同的构建获取不同的更新。

<ContentSpotlight
  alt='渠道 "production" 链接到分支 "version-1.0"'
  src="/static/images/eas-update/channel-branch-link.png"
/>

上面的图示展示了这种链接关系。这里，我们有一个构建，其 "production" 渠道链接到了名为 "version-1.0" 的分支。准备好之后，我们可以调整渠道–分支指针。设想我们在名为 "version-2.0" 的分支上还有更多已测试并准备好的修复。我们可以更新这个链接，让 "version-2.0" 分支对所有使用 "production" 渠道的构建可用。

<ContentSpotlight
  alt='渠道 "production" 链接到分支 "version-2.0"'

/>

## 检查项目更新的状态

### 查看渠道

查看所有渠道：

<Terminal cmd={['$ eas channel:list']} />

查看特定渠道：

<Terminal
  cmd={['$ eas channel:view [channel-name]', '', '# 示例', '$ eas channel:view production']}
  cmdCopy="eas channel:view [channel-name]"
/>

创建渠道：

<Terminal
  cmd={['$ eas channel:create [channel-name]', '', '# 示例', '$ eas channel:create production']}
  cmdCopy="eas channel:create [channel-name]"
/>

### 查看分支

查看所有分支：

<Terminal cmd={['$ eas branch:list']} />

查看特定分支及其更新列表：

<Terminal
  cmd={['$ eas branch:view [branch-name]', '', '', '# 示例', '$ eas branch:view version-1.0']}
  cmdCopy="eas branch:view [branch-name]"
/>

### 查看更新

查看特定更新：

<Terminal
  cmd={[
    '$ eas update:view [update-group-id]',
    '',
    '# 示例',
    '$ eas update:view dbfd479f-d981-44ce-8774-f2fbcc386aa',
  ]}
  cmdCopy="eas update:view [update-group-id]"
/>

## 更改项目更新的状态

### 创建并发布一个新更新

<Terminal
  cmd={[
    '$ eas update --branch [branch-name] --message "..."',
    '',
    '# 示例',
    '$ eas update --branch version-1.0 --message "Fixes typo"',
  ]}
  cmdCopy="eas update --branch [branch-name] --message"
/>

如果你正在使用 Git，我们可以使用 `--auto` 标志自动填充分支名称和消息。这个标志会使用当前 Git 分支作为分支名称，并使用最新的 Git 提交信息作为消息。

<Terminal cmd={['$ eas update --auto']} />

### 删除分支

<Terminal
  cmd={['$ eas branch:delete [branch-name]', '', '# 示例', '$ eas branch:delete version-1.0']}
  cmdCopy="eas branch:delete [branch-name]"
/>

### 重命名分支

重命名分支不会断开任何渠道–分支链接。如果你有一个名为 "production" 的渠道链接到名为 "version-1.0" 的分支，然后你将名为 "version-1.0" 的分支重命名为 "version-1.0-new"，那么 "production" 渠道将会链接到现在已重命名的分支 "version-1.0-new"。

<Terminal
  cmd={[
    '$ eas branch:rename --from [branch-name] --to [branch-name]',
    '',
    '# 示例',
    '$ eas branch:rename --from version-1.0 --to version-1.0-new',
  ]}
  cmdCopy="eas branch:rename --from [branch-name] --to [branch-name]"
/>

### 在分支内重新发布之前的更新

我们可以让之前的更新立即对所有用户可用。此命令会获取之前的更新并再次发布，使其成为该分支上的最新更新。当用户重新打开应用时，应用会看到重新发布后的更新并下载它。

> 重新发布类似于 Git 反转，其中正确的提交会被放到 Git 历史记录的顶部。

<Terminal
  cmd={[
    '$ eas update:republish --group [update-group-id]',
    '$ eas update:republish --branch [branch-name]',
    '',
    '# 示例',
    '$ eas update:republish --group dbfd479f-d981-44ce-8774-f2fbcc386aa',
    '$ eas update:republish --branch version-1.0',
  ]}
/>

> 如果你不知道确切的更新组 ID，可以使用 `--branch` 标志。这会显示该分支上最近更新的列表，并允许你选择要重新发布的更新组。
