> **警告** 由于 `react-native-screens` 的一个上游问题，当前单一路由会被标记为 `dangerouslySingular`。Expo 正在与 Software Mansion 合作解决该问题。

虽然 `Stack` 通常通过 `push`/`pop` 操作运行，但在高级场景中，你可能希望在堆栈中强制某个屏幕只能有单一版本。启用后，每次推入新屏幕时，都会扫描历史记录并移除任何匹配的屏幕，以确保只存在一个单一版本。

## 选项

条件有两个选项：

- `true`：使用默认的单一函数
- `Function`：指定你自己的单一函数

默认的单一函数会返回屏幕的 pathname，不包含查询参数或 hash。

单一标识符函数的签名为 `(routeName: string, params: Record<string, string | string[]>) => string | undefined`。

## 在 navigator 上设置单一

<FileTree files={['app/_layout.tsx', 'app/index.tsx', 'app/[profile].tsx']} />

使用上面的示例应用，你可以创建一个单一约束，以确保每个 `[profile]` 页面都是唯一的。

```tsx app/_layout.tsx
export default function () {
  return (
    <Stack>
      <Stack.Screen name="[profile]" dangerouslySingular={({ profile }) => profile} />
    </Stack>
  );
}
```

如果你现在导航到 `/profile-2`，并且你的历史记录是：

- `/profile-1`
- `/profile-2`
- `/profile-3`

新的历史记录会推入新路由，同时也会移除任何之前的条目：

- `/profile-1`
- `/profile-3`
- `/profile-2`

## 创建单一路由链接

`dangerouslySingular` 也可以添加到 `<Link />` 组件中，以创建强制单一路由的链接。

```tsx
<Link href="/unique-link" dangerouslySingular />
<Link href="/unique-link" dangerouslySingular={(name) => name === 'unique-link' ? name : undefined} />
```

## 使用命令式 API

命令式 API 也接受 `dangerouslySingular` 选项。

```tsx
router.navigate('/unique-link', { dangerouslySingular: true });
router.navigate('/unique-link', {
  dangerouslySingular: name => (name === 'unique-link' ? name : undefined),
});

router.push('/unique-link', { dangerouslySingular: true });
router.push('/unique-link', {
  dangerouslySingular: name => (name === 'unique-link' ? name : undefined),
});
```

## navigate vs push

`navigate` 和 `push` 事件以不同方式应用单一约束。

- _navigate_：只有在当前路由发生变化时才应用该约束
- _push_：总是应用该约束

<FileTree files={['app/_layout.tsx', 'app/index.tsx', 'app/[profile].tsx']} />

使用上面的应用结构，如果你的历史记录是：

- `/profile-1`
- `/profile-2`
- `/profile-2`
- `/profile-3`
- `/profile-2`

如果你导航到 `/profile-2`，由于使用 `navigate` 时当前路由已经是目标路由，因此不会应用单一约束。然而，如果你 push 到 `/profile-2`，则会应用单一约束。

```tsx
// 当当前路由就是目标路由时，不会应用单一约束
router.navigate("/profile-2", {
  dangerouslySingular: (_, { profile } => profile === 'profile-2' ? profile : undefined )
})

// 将会应用单一约束
router.push("/profile-2", {
  dangerouslySingular: (_, { profile } => profile === 'profile-2' ? profile : undefined )
})
```
